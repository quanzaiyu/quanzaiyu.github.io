      # 使用 vConsole 进行移动端调试

      在移动端开发有一个很麻烦的问题：调试。

      在 Android 还好，但是在 iOS 进行调试还是异常痛苦，虽然可以使用 Fiddler 设置代理进行抓包调试，但是，很多程序上的问题并不能进行有效的调试，不像 PC 控制台那样，直接看得到输出。

      Android 在应用程序上可以嵌入厂商自己研制的浏览器，比如微信使用的就是 X5 内核的浏览器，而 iOS 就比较撮了，强制使用系统本身的 safari，导致很多问题不但程序起来兼容困难，而且调试也困难。

      以前的测试，很多时候我会 alert 出来进行测试，不过对于一些比较大的对象（比如 File、Blob）就既不好观察也不好调试。

      如果是 Android 微信端，可以直接使用 http://debugx5.qq.com 进行配置调试。而 iOS，其实也可以安装 vConsole 进行调试。

      # 安装 vConsole

      此项目的 GitHub 地址为：https://github.com/Tencent/vConsole。

      安装：

      yarn add vconsole

      我是在 Vue 中使用 vConsole 的，直接在 main.js 中初始化 vConsole 就好了：

      import Vconsole from 'vconsole'

/* eslint-disable no-unused-vars */
if (/(testapp|localhost|192\.168)/.test(window.location.hostname)) {
  const vConsole = new Vconsole()
}

      上面之所以加了一个条件，意思是只需要在开发环境和测试环境添加即可。

      # 直接使用

      虽然我没在 HTML 中直接引用，但是还是将官网中的相关代码引入，以供参考：

      <script src="path/to/vconsole.min.js"></script>
<script>
  var vConsole = new VConsole();
</script>
