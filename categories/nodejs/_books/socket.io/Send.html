# 单发消息

单发消息有两种方式:

  • socket.send
  • socket.emit

send 其实是 emit('message') 的封装:

function send(){
  var args = toArray(arguments);
  args.unshift('message');
  this.emit.apply(this, args);
  return this;
}

emit函数有两个参数

  • 第一个参数是自定义的事件名称,发送方发送什么类型的事件名称,接收方就可以通过对应的事件名称来监听接收
  • 第二个参数是要发送的数据

# 客户端向服务器端发送消息

socket.send('hello')
socket.emit('chat message', 'hello')

在服务器端通过对应的事件实现监听

socket.on('message', msg => {
  console.log(socket.id);
  console.log(msg)
})
socket.on('chat message', function(msg){
  console.log(socket.id);
  console.log(msg);
});

socket.id 是socket的一个属性，存着这次socket连接的id，是唯一标识的，我们实现私聊就可以通过该id找到用户。

# 服务器端向客户端发送消息

同样地, 服务器端也可通过上述方法实现消息的发送, 在收到客户端传来的 message 时作出回应:

socket.on('message', msg => {
  socket.send(`服务器回应：welcome`);
  socket.emit('new user', '欢迎加入');
})

客户端同样实现对应的监听即可:

socket.on('message', function(msg){
  console.log(msg);
});
socket.on('new user', function(msg){
  console.log(msg);
});

# 向某一用户发送消息

可以通过 socket.id 确定该用户的唯一标识, 实际应用中可以将用户信息绑定 socket.id 存入redis, 比如形成以下组合信息:

{
  userId: 1000,
  socketId: 'ZNGlLFQLEf17DC07AAAJ',
  name: '小昱',
  avatar: ...
}

客户端只要知道该用户的 userId, 服务器端即可确定其 socketId, 然后通过 socket.to(socketId) 向此用户发送消息
