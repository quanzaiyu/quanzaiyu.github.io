        # 关于小程序 textarea 各种问题的解决方案

        # 不随页面滚动的解决方案

        在使用小程序的时候, 出现textarea中的文字不随组件滚动而滚动的问题, 现象:

        经过对官方文档的阅读, 以及一系列排查, 发现 textarea 属于原生组件, 不能放于 scroll-view 组件中, 或者是带 overflow: scroll; 属性的元素中, 有的话将此属性删除, 或换其他方式实现。

        # 层级过高的处理方案

        这是小程序最大的 bug, 调这个 bug 也费了很大的精力和时间, 无论浮层元素层级多高, 即使是 z-index 为 999, 原生组件仍然会覆盖到浮层之上, 特别是表单元素, 内容或者 placeholder 都会在最上层。

        目前有一种还算可行的解决方案:

        在文本框焦点失去时, 隐藏文本框, 并显示一个样式一模一样的 view 置于相同位置。当点击此 view 是, view 消失, 并使文本框获取到焦点:

        <view class="module-container">
  <view class="remarks-container">
    <view class="remarks-title">备注</view>
    <view class="remarks-wrap">
      <textarea wx:if="{{!isShowText}}" class="remarks" bindblur="onRemarkInput" bindconfirm="onShowText" focus="{{onFocus}}" maxlength="100" value="{{remark}}"/>
      <text wx:else class="remarks" bindtap="onShowTextarea">{{remark}}</text>
    </view>
  </view>
</view>

        page({
  data: {
    onFocus: false,    //textarea焦点是否选中
    isShowText：false, //控制显示 textarea 还是 text
    remark: '',        //用于存储textarea输入内容
  },
  onShowTextare() {       //显示textare
    this.setData({
      isShowText: false,
      onFacus: true
    })
  },
  onShowText() {       //显示text
    this.setData({
      isShowText: true,
      onFacus: false
    })
  },
  onRemarkInput(event) {               //保存输入框填写内容
    var value = event.detail.value;
    this.setData({
      remark: value,
    });
  },
})

        # 一些限制

        textarea 组件使用时有以下限制:

        • bug: 微信版本 6.3.30，textarea 在列表渲染时，新增加的 textarea 在自动聚焦时的位置计算错误。
        • tip: textarea 的 blur 事件会晚于页面上的 tap 事件，如果需要在 button 的点击事件获取 textarea，可以使用 form 的 bindsubmit。
        • tip: 不建议在多行文本上对用户的输入进行修改，所以 textarea 的 bindinput 处理函数并不会将返回值反映到 textarea 上。

        另外, 附上原生组件的使用限制:

        • 原生组件包括: camera canvas input live-player live-pusher map textarea video
        • 原生组件脱离在 WebView 渲染流程外
        • 原生组件的层级是最高的，所以页面中的其他组件无论设置 z-index 为多少，都无法盖在原生组件上。
        • 后插入的原生组件可以覆盖之前的原生组件。
        • 原生组件还无法在 scroll-view、swiper、picker-view、movable-view 中使用。
        • 部分CSS样式无法应用于原生组件，例如：
          • 无法对原生组件设置 CSS 动画
          • 无法定义原生组件为 position: fixed
          • 不能在父级节点使用 overflow: hidden 来裁剪原生组件的显示区域
        • 原生组件的事件监听不能使用 bind:eventname 的写法，只支持 bindeventname。原生组件也不支持 catch 和 capture 的事件绑定方式。
        • 在iOS下，原生组件暂时不支持触摸相关事件。
        • 原生组件会遮挡 vConsole 弹出的调试面板。
        • 在工具上，原生组件是用web组件模拟的，因此很多情况并不能很好的还原真机的表现，建议开发者在使用到原生组件时尽量在真机上进行调试。
