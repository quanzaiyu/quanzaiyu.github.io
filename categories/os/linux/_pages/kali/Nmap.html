# 安装Nmap

apt Install nmap

# 扫描指定网段

nmap -sn 192.168.126.1-200

# 跳过Ping扫描阶段 (无Ping扫描), 列出启用的端口

通常Nmap在进行其他扫描之前, 都会对目标进行一个Ping扫描, 如果目标对Ping扫描没有反应的话, 就会直接结束整个扫描阶段 (服务器可以禁止Ping)

nmap -PN <ip>

# 仅使用Ping协议进行主机发现

有的时候, 需要快速的大量的足迹中去快速发现主机, 这个时候, 仅使用Ping协议进行扫描, 速度会非常快

nmap -sP <ip>

# 使用ARP协议进行主机发现

ARP协议扫描仅适用于局域网内, 使用ARP, 不经速度快, 而且结果也会非常准确

nmap -PR <ip>

# 半开扫描与全开扫描

TCP协议三次握手

 ----  1. SYN报文   ------->

黑客 <--- 2. SYN+ACK报文 ------ 目标 ---- 3. ACK报文 ------->

3个过程全部完成叫全开扫描

最后一步不做, 叫做半开扫描

nmap -sS <ip> # 半开扫描
nmap -sT <ip> # 全开扫描

在实际应用中, 半开扫描应用得最多, 半开扫描不容易被目标主机日志记录

# 识别操作系统

因为操作系统不同, 渗透方式也不同, 所以判断目标电脑使用的操作系统非常重要

nmap -O <ip>

# 端口发现

实际主机上可以运行65536个端口, nmap默认扫描其中最常用的1000个端口

扫描全部端口

nmap -p "*" <ip>

扫描指定端口

nmap -p 80 <ip>
