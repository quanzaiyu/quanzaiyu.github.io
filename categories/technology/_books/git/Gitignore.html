# gitignore

如果不希望 git 系统追踪某些文件, 可以配置 .gitignore 文件

文件 .gitignore 的格式规范如下：

  • 所有空行或者以 ＃ 开头的行都会被 Git 忽略。
  • 可以使用标准的 glob 模式 (shell 所使用的简化了的正则表达式) 匹配。
  • 匹配模式可以以（/）开头防止递归。
  • 匹配模式可以以（/）结尾指定目录。
  • 要忽略指定模式以外的文件或目录，可以在模式前加上惊叹号（!）取反。

# 取消文件跟踪

如果不小心 add 了不需要跟踪的文件, 之后配置 .gitignore 是不生效的, 可以使用 git rm 移除文件跟踪, 注意, 如果不加 --cache 的话则

$ git rm <FileName> # 此操作将从本地版本和磁盘中删除
$ git rm --cached -r dist # 移除dist目录下的所有文件跟踪(不从磁盘删除)
