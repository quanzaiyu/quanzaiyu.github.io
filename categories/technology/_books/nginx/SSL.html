# 配置ssl证书

要让网站拥有 HTTPS (Hypertext Transfer Protocol Secure) 的能力, 需要在服务器中配置 SSL (Secure Socket Layer) 证书。

# 证书申请

首先是证书的申请, 有好几种方式, 很多证书是要收费的, 我们可以使用阿里云云盾服务进行证书申请。

或者腾讯云进行申请

# 在Nginx中配置SSL

下载下来的文件包括:

注意到有一个 pem 文件和一个 key 文件, 分别将其路径配置到 ssl_certificate 和 ssl_certificate_key 字段, 比如:

server {
    listen       443 ssl http2 default_server;
    listen       [::]:443 ssl http2 default_server;
    server_name  _;
    root         /usr/share/nginx/html;

    ssl_certificate "/root/.ssl/ssl.pem";
    ssl_certificate_key "/root/.ssl/ssl.key";
    ssl_session_cache shared:SSL:1m;
    ssl_session_timeout  10m;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # Load configuration files for the default server block.
    # include /etc/nginx/default.d/*.conf;

    location / {
    }

    error_page 404 /404.html;
        location = /40x.html {
    }

    error_page 500 502 503 504 /50x.html;
        location = /50x.html {
    }
}

如果是 crt 证书, 同样的配置:

ssl_certificate "/root/.ssl/ssl.crt";
ssl_certificate_key "/root/.ssl/ssl.key";

重新加载配置:

$ nginx -s reload

在防火墙中打开指定端口:

$ firewall-cmd --add-port=443/tcp

ok, 一切就绪, 浏览器中访问试试:
