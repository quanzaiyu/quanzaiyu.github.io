          # 个人博客中引入 Gitment 评论系统

          # 前言

          以前是用 PHP 搭建的动态网站，使用的多说评论系统。不过，早在一年前，多说就宣布关闭了。那个时候，正想换一套博客系统，不过也蛮折腾的。

          开始选用 Jekyll，后来到 Hexo，都不是特别满意，用了一段时间就不想用了。

          直到尤大大的 Vuepress 发布。

          也就是当前我使用的这套系统。想着要弄就好好弄一个个人网站，原本 Vuepress 已经非常完善了，但是还差一些东西，比如评论系统。

          在网络上四处寻觅，一一排除了网易云跟贴、畅言等几种类似的社会化评论系统。原因很简单，我不敢确定这些服务的稳定性，万一什么时候跟多说一样说关就关，那又得重新折腾。

          最后发现两款比较好用的项目：comment.js 和 gitment，最终选择了 gitment，由于 comment.js 不能使用 npm 进行管理，而我的项目又是一个使用 Vuepress 搭建的应用，当然更加侧重于 gitment。

          不过两个项目都有一个共同点，使用 GitHub 的 issue 来搭建博客的评论系统。目前，comment.js 还支持使用 gitee 的 issue 系统来搭建评论系统。

          相关链接：comment.js - GitHub、gitment - GitHub

          # 使用

          1. 注册 OAuth Application

          打开 https://github.com/settings/applications/new 注册一个 OAuth Application，成功后记住 Client ID和Client Secret。

          1. 引入 Gitment

          在需要的页面引入：

          <div id="container"></div>
<link rel="stylesheet" href="https://imsun.github.io/gitment/style/default.css">
<script src="https://imsun.github.io/gitment/dist/gitment.browser.js"></script>
<script>
var gitment = new Gitment({
  id: '页面 ID', // 可选。默认为 location.href
  owner: '你的 GitHub ID',
  repo: '存储评论的 repo',
  oauth: {
    client_id: '你的 client ID',
    client_secret: '你的 client secret',
  },
})
gitment.render('container')
</script>

          如果是Vuepress，可以在 Page.vue 中引入：

          首先安装 yarn add gitment

          然后引入：

          <template lang="pug">
  .page
  	// ...
    #comments
</template>

<script>
import 'gitment/style/default.css'
import Gitment from 'gitment'
export default {
  // ...
  mounted () {
    const gitment = new Gitment({
      id: this.$page.title,
      owner: 'quanzaiyu',
      repo: 'xiaoyulive-comment',
      oauth: {
        client_id: 'xxx',
        client_secret: 'xxx',
      },
    })
    gitment.render(document.getElementById('comments'))
  }
}
</script>

          具体的参数常见：Gitment Options

          1. 初始化评论

          页面发布后，你需要访问页面并使用你的 GitHub 账号登录（请确保你的账号是第二步所填 repo 的 owner），点击初始化按钮。

          之后其他用户即可在该页面发表评论。

          1. 自定义样式

          Gitment 很容易进行自定义，你可以写一份自定义的 CSS 或者使用一个新的主题。（主题可以改变 DOM 结构而自定义 CSS 不能）

          比如你可以通过自定义主题将评论框放在评论列表前面：

          const myTheme = {
  render(state, instance) {
    const container = document.createElement('div')
    container.lang = "en-US"
    container.className = 'gitment-container gitment-root-container'
    container.appendChild(instance.renderHeader(state, instance))
    container.appendChild(instance.renderEditor(state, instance))
    container.appendChild(instance.renderComments(state, instance))
    container.appendChild(instance.renderFooter(state, instance))
    return container
  },
}
const gitment = new Gitment({
  // ...
  theme: myTheme,
})
gitment.render('container')

          更多自定义内容请查看文档。

          # 常见问题

          # Error: Comments Not Initialized

          • 在注册OAuth Application这个步骤中，给 Authorization callback URL 指定的地址错了
          • 还没有在该页面的 Gitment 评论区登陆 GitHub 账号

          # 不要在本地测试

          如果在本地初始化时在评论框点击登录，但是却没有跳到授权页面，因为在 setting - OAuth Application 注册页面填的 Authorization Callback URL 是自己的域名，不是本地调试的域名，比如 http://localhost:4000/。当然，蛋疼的话可以另外申请一个 OAuth Application 填 http://localhost:4000/ 也是可以的。

          # 422 Error：validation failed

          原因是 id 不合法，比如默认使用了 location.href，这样的话可能会因为路径太长而造成无法通过验证。

          本人使用的是 Vuepress，因此在 Page.vue 里面配置了 id: this.$page.title。
