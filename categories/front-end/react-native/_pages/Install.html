      # 环境搭建

      安装脚手架

      $ yarn global add react-native-cli

      创建项目

      $ react-native init <ProjectName>

      # 运行到 Android

      在 Windows 下, 首先需要配置环境变量 ANDROID_HOME, 配置 Android SDK 路径, 比如 C:\Users\quanz\AppData\Local\Android\Sdk (可在 Android Studio 的配置里面看到 File > Settings > Appearance & Behavior > System Settings > Android SDK)

      终端输入 react-native run-android 运行到真机或模拟器

      可以看到成功运行到模拟器:

      在 Development 模式下, js 部分有 Errors 或 Warning, 会直接打印在手机或模拟器屏幕上, 以红屏或黄屏展示。

      # 运行到 iOS

      在 MacOS 下, 安装 XCode, 终端输入 react-native run-ios 即可

      # 调试

      # 热加载

      要开启热加载, 在模拟器中按 Ctrl + M 进行设置, 在真机上通过摇一摇的方式打开开发者菜单:

      如果需要手动刷新, 直接在模拟器中按 RR 即可

      • Enable Live Reload 有变更重新加载整个项目, 页面回到初始页
      • Enable Hot Reloading 有变更重新加载到当前页面

      # Chrome Debug

      在开发者菜单中选择 Debug JS Remotely, 可以在 Chrome 中进行调试, 可以看到 Chrome 浏览器打开了调试地址 http://localhost:8081/debugger-ui/

      # react-native-debugger

      使用 react-native-debugger 可以更好地调试 React Native 应用:

      在模拟器开发者菜单中使用 Toggle Inspector 可以监控页面里面的组件, 跟 Chrome 中 Elements 选项一样。

      # 访问控制台日志

      使用以下命令可以访问控制台命令:

      $ react-native log-ios
$ react-native log-android

      参考: 调试

      # 控制台打印

      跟 web 一样, 可以使用 console 在控制台进行输出:

      • console.log()
      • console.warn()
      • console.error()
