        # Redux 核心思想

        # 三大原则

        1. 单一数据源

        整个应用的 state 被储存在一棵 object tree 中，并且这个 object tree 只存在于唯一一个 store 中。

        1. State 是只读的

        唯一改变 state 的方法就是触发 action，action 是一个用于描述已发生事件的普通对象。

        1. 使用纯函数来执行修改

        为了描述 action 如何改变 state tree ，你需要编写 reducers。

        # Action

        Action 是把数据从应用（译者注：这里之所以不叫 view 是因为这些数据有可能是服务器响应，用户输入或其它非 view 的数据 ）传到 store 的有效载荷。它是 store 数据的唯一来源。一般来说你会通过 store.dispatch() 将 action 传到 store。

        如下便是一个 action:

        {
  type: 'SET_APP_NAME',
  text: 'Redux app'
}

        发起此 action 需要使用 dispatch:

        store.dispatch({
  type: 'SET_APP_NAME',
  text: 'Redux app'
})

        Action 本质上是 JavaScript 普通对象。我们约定，action 内必须使用一个字符串类型的 type 字段来表示将要执行的动作。多数情况下，type 会被定义成字符串常量。当应用规模越来越大时，建议使用单独的模块或文件来存放 action。

        actionTypes.js

        const SET_APP_NAME = 'SET_APP_NAME'

        引入 actionTypes.js

        import { SET_APP_NAME } from '../actionTypes'

...

store.dispatch({
  type: SET_APP_NAME,
  text: 'Redux app'
})

        # Action 创建函数

        上面的做法, 我们通过 dispatch 直接调用 action, 我们还可以使用 Action 创建函数 进行处理后调用, 所谓 Action 创建函数 就是生成 action 的方法。

        比如:

        function setAppName(text) {
  return {
    type: SET_APP_NAME,
    text
  }
}

...

store.dispatch(setAppName('Redux app'))

        # Reducer

        Reducer 指定了应用状态的变化如何响应 actions 并发送到 store 的，记住 actions 只是描述了有事情发生了这一事实，并没有描述应用如何更新 state。

        需要谨记 reducer 一定要保持纯净。只要传入参数相同，返回计算得到的下一个 state 就一定相同。没有特殊情况、没有副作用，没有 API 请求、没有变量修改，单纯执行计算。

        比如:

        const app = (state = "My app", action) => {
  switch (action.type) {
    case 'SET_APP_NAME':
      return action.text
    default:
      return state
  }
}

        注意事项:

        1. 不要修改 state。 使用 Object.assign({}, state, { newState: action.newState }) 新建了一个副本。不能这样使用 Object.assign(state, { newState: action.newState })，因为它会改变第一个参数的值。你必须把第一个参数设置为空对象。你也可以开启对ES7提案对象展开运算符的支持, 从而使用 { ...state, ...newStates } 达到相同的目的。
        2. 在 default 情况下返回旧的 state。遇到未知的 action 时，一定要返回旧的 state。

        # Reducer 组合

        如果有多个 Reducer, Redux 提供了 combineReducers() 工具类来组合多个 Reducer。

        比如:

        import { combineReducers } from 'redux'

import app from './modules/app'

// 有多个 Reducer 可继续往里面添加
const App = combineReducers({
  app
})

export default App

        以下写法同上面完全等价:

        import app from './modules/app'

export default function App(state = {}, action) {
  return {
    app: app(state.app, action)
  }
}

        # Store

        Store 就是把它们联系到一起的对象。Store 有以下职责：

        • 维持应用的 state
        • 提供 getState() 方法获取 state
        • 提供 dispatch(action) 方法更新 state
        • 通过 subscribe(listener) 注册监听器
        • 通过 subscribe(listener) 返回的函数注销监听器

        ::: warn 注意 Redux 应用只有一个单一的 store。当需要拆分数据处理逻辑时，你应该使用 reducer 组合 而不是创建多个 store。 :::

        使用上面组合的 reducer 创建一个 store:

        import combinedReducer from "./reducer"

import {createStore} from 'redux'

export default createStore(combinedReducer)

        # subscribe

        通过 subscribe 可以在更新 state 时调用, 主要用于同步视图等动作。

        示例

        ...

// 每次 state 更新时，打印日志
// 注意 subscribe() 返回一个函数用来注销监听器
const unsubscribe = store.subscribe(() =>
  console.log(store.getState())
)

...

// 停止监听 state 更新
unsubscribe();
