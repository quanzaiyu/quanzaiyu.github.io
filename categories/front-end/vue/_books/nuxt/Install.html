# 安装和使用

# Nuxt 安装

使用脚手架 create-nuxt-app 进行安装:

$ npx create-nuxt-app <项目名>
# or
$ yarn create nuxt-app <项目名>

安装完成后, 使用 npm run dev 运行, 默认开启的路径是 localhost:3000

Nuxt.js 会监听 pages 目录中的文件更改，在添加新页面时无需重新启动应用程序。

  • 构建应用程序: yarn build
  • 开启应用程序: yarn start

安装 nuxt 全局包以便直接执行 nuxt 命令(非必须):

$ yarn global add nuxt

# 目录结构

  • pages 页面目录, 用于组织应用的路由及视图。Nuxt.js 框架读取该目录下所有的 .vue 文件并自动生成对应的路由配置。该目录名为Nuxt.js保留的，不可更改。
  • layouts 布局目录, 用于组织应用的布局组件。该目录名为Nuxt.js保留的，不可更改。
  • static 静态文件目录, 用于存放应用的静态文件，此类文件不会被 Nuxt.js 调用 Webpack 进行构建编译处理。服务器启动的时候，该目录下的文件会映射至应用的根路径 / 下。该目录名为Nuxt.js保留，不可更改。
  • store 目录用于组织应用的 Vuex 状态树 文件。 Nuxt.js 框架集成了 Vuex 状态树 的相关功能配置，在 store 目录下创建一个 index.js 文件可激活这些配置。该目录名为Nuxt.js保留，不可更改。
  • assets 资源目录, 用于组织未编译的静态资源如 LESS、SASS 或 JavaScript。
  • components 组件目录, 用于组织应用的 Vue.js 组件。Nuxt.js 不会扩展增强该目录下 Vue.js 组件，即这些组件不会像页面组件那样有 asyncData 方法的特性。
  • plugins 插件目录, 用于组织那些需要在 根vue.js应用 实例化之前需要运行的 Javascript 插件。
  • middleware 中间件目录。
  • nuxt.config.js 文件用于组织Nuxt.js 应用的个性化配置，以便覆盖默认配置。该文件名为Nuxt.js保留的，不可更改。
  • package.json 文件用于描述应用的依赖关系和对外暴露的脚本接口。该文件名为Nuxt.js保留的，不可更改。

# 别名

  • ~ 或 @: src目录
  • ~~ 或 @@: 根目录
  • 默认情况下，src目录 和 根目录 相同

提示

在 vue 模板中, 如果你需要引入 assets 或者 static 目录, 使用 ~/assets/your_image.png 和 ~/static/your_image.png 方式。

# 部署

  • nuxt 启动一个热加载的Web服务器（开发模式） localhost:3000。
  • nuxt build 利用webpack编译应用，压缩JS和CSS资源（发布用）, 生成路径为 .nuxt/dist。
  • nuxt start 以生成模式启动一个Web服务器 (nuxt build 需先执行)。
  • nuxt generate 编译应用，并依据路由配置生成对应的HTML文件 (用于静态站点的部署), 生成路径为 dist。

生成 spa:

  1. 将 nuxt.config.js 中的 mode 更改为 spa。
  2. 运行 npm run build.
  3. 自动生成 dist/ 文件夹，部署到您的服务器，如 Surge，GitHub Pages或nginx。
