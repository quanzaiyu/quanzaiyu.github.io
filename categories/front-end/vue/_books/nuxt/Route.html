# 路由

# 路由自动解析

在 pages 目录下的 vue 文件, 将会根据路径自动解析为路由。

比如有如下结构

pages/
--| user/
-----| index.vue
-----| one.vue
--| index.vue
--| test.vue

生成的路由如下:

router: {
  routes: [
    {
      name: 'index',
      path: '/',
      component: 'pages/index.vue'
    },
    {
      name: 'test',
      path: '/test',
      component: 'pages/test.vue'
    },
    {
      name: 'user',
      path: '/user',
      component: 'pages/user/index.vue'
    },
    {
      name: 'user-one',
      path: '/user/one',
      component: 'pages/user/one.vue'
    }
  ]
}

在 index.vue 中使用以下语法可以执行路由:

nuxt-link(to="/test") 测试
nuxt-link(to="/user") 用户首页
nuxt-link(to="/user/one") 用户one

也可以以对象的形式传递:

nuxt-link(:to="{name:'user'}") user
nuxt-link(:to="{name:'user-one'}") user

注意

如果是在目录下的路由, 生成的 name 由 - 进行连接

也可使用脚本控制跳转:

<template lang='pug'>
.container
  div(@click='go') go
</template>

<script>
export default {
  methods: {
    go() {
      this.$router.push('/user')
      // or 对象写法
      // this.$router.push({ path: '/user' })
    }
  }
}
</script>

相关路由 API 来源于 vue-router.

# 嵌套路由与动态路由

  • 创建子路由，需要在父路径添加一个与子路由目录同名的 Vue 文件
  • 创建动态路由，需要创建对应的以下划线 _ 作为前缀的 Vue 文件 或 目录。
pages/
--| users/
-----| index.vue
-----| _id.vue
--| index.vue
--| users.vue

生成的路由如下:

router: {
  routes: [
    {
      name: 'index',
      path: '/',
      component: 'pages/index.vue'
    },
    {
      path: '/users',
      component: 'pages/users.vue',
      children: [
        {
          path: '',
          component: 'pages/users/index.vue',
          name: 'users'
        },
        {
          path: ':id',
          component: 'pages/users/_id.vue',
          name: 'users-id'
        }
      ]
    }
  ]
}

使用动态路由:

<template lang='pug'>
.container
  div(@click='go(1)') go1
  div(@click='go("xiaoyu")') go2
</template>

<script>
export default {
  methods: {
    go(id) {
      this.$router.push({
        path: `/users/${id}`
      })
    }
  }
}
</script>

如果想将它设置为必选的路由，需要在 users/_id 目录内创建一个 index.vue 文件。

在 pages/users/_id.vue, 获取路由参数:

.container hello {{$route.params.id}}

# 路由参数校验

在 pages/users/_id.vue, 通过 validate 验证路由参数的有效性:

export default {
  validate({ params }) {
    // 必须是number类型
    return /^\d+$/.test(params.id)
  }
}

如果校验方法返回的值不为 true或Promise中resolve 解析为false或抛出Error ， Nuxt.js 将自动加载显示 404 错误页面或 500 错误页面。

以上代码, 如果路由参数不为数字, 则报 404 错误。

# 全局路由过渡动画

在 assets/ 下添加 main.css 文件, 键入:

.page-enter-active, .page-leave-active {
  transition: opacity .5s;
}
.page-enter, .page-leave-active {
  opacity: 0;
}

在 nuxt.config.js 中配置:

module.exports = {
  css: [
    'assets/main.css'
  ]
}

这将应用到全局, 所有页面的过渡都会执行此过渡动画。

# 页面路由过渡动画

如果只是想要给某个页面自定义过渡特效的话，只要在该页面组件中配置 transition 字段即可:

<template lang='pug'>
.container hello world
</template>

<script>
export default {
  transition: 'test'
}
</script>

<style lang='stylus' scoped>
.test-enter-active, .test-leave-active {
  transition: opacity .5s;
}
.test-enter, .test-leave-active {
  opacity: 0;
}
</style>
