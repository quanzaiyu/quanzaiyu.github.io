# 响应式的数据

阅读过 Vue 源码后，对其响应式原理也有了一定理解，于是自己试图搭建一个简单的响应式脚本。

# 约定观测对象

首先，结合我们平常写 Vue 的方式：

const vue = new Vue({
  data: {
    a: 1
  }
})

vue.$watch('a', () => {
  console.log('修改了 a')
})

我们约定我们需要观测的对象为：

const data = {
  a: 1
}

我们使用的观测方法为：

$watch('a', () => {
  console.log('a changed')
})

这里的关键，就是如何实现观测方法 $watch。

# 数据监测

要实现观测函数，说简单也简单，所困难也困难。

之所以说简单，如果我们不考虑观测的临界条件以及一些特殊情况的处理，其实是非常简单的，使用 Object.defineProperty 方法，我们可以定义某个对象的某些属性的 setter 和 getter：

Object.defineProperty(data, 'a', {
  set () {
    console.log('set a')
  },
  get () {
    console.log('get a')
  }
})

这样的话，我们在控制台对 a 进行读写的时候，都会在控制台打印相关的语句。

但是实际上，我们还需要考虑的问题远远不止这些，比如对象的循环遍历，试图的同步渲染等等。所以说，也不容易。

# 依赖搜集

当然，在观测到 a 变化的时候，我们肯定希望自定义进行处理，而不是像上面代码一样直接在 set 和 get 中打印一段死文字。

因此，我们需要从外部传递依赖，这里的思路是：在获取属性 a 的时候收集依赖，然后在设置属性 a 的时候触发之前收集的依赖。

我们使用一个容器 dep 搜集依赖：

const dep = []
Object.defineProperty(data, 'a', {
  set () {
    // 当属性被设置的时候，将容器里的依赖都执行一次
    dep.forEach(fn => fn())
  },
  get () {
    // 当属性被获取的时候，把依赖放到容器里
    dep.push(fn)
  }
})

那如何在获取属性 a 的值时收集依赖呢？

注意到我们的观测方法是 $watch，那么，就在这个观测方法中手动触发 a 的 get：

const dep = []
Object.defineProperty(data, 'a', {
  set () {
    dep.forEach(fn => fn())
  },
  get () {
    // 此时 Target 变量中保存的就是依赖函数
    dep.push(Target)
  }
})

// Target 是全局变量
let Target = null
function $watch (exp, fn) {
  // 将 Target 的值设置为 fn
  Target = fn
  // 读取字段值，触发 get 函数
  data[exp]
}

以上，就完成了依赖搜集的工作。这事，在控制台中改变 data.a 的值，可以发现控制台打印出了 a changed。

同样的，我们可以添加多个依赖：

$watch('a', () => {
  console.log('第一个依赖')
})
$watch('a', () => {
  console.log('第二个依赖')
})

这样的话，改变 data.a，多个依赖会依次执行。

# 观测多个值

上面的实现，只是能够观测到属性 a，但是，如果我们在 data 中添加其他属性，则就不起效了：

const data = {
  a: 1,
  b: 1
}

// ...

$watch('b', () => {
  console.log('b changed')
})

为了能够观测 data 下面的所有属性，我们可以使用一个循环将容器包裹住，依次观测 data 下面的所有属性：

for (const key in data) {
  const dep = []
  Object.defineProperty(data, key, {
    set () {
      dep.forEach(fn => fn())
    },
    get () {
      dep.push(Target)
    }
  })
}

# 赋值取值

上面的代码，虽然能够观测到 data 下面的所有属性了，但是，实际上并不会对 data 下面的属性进行赋值，也获取不到任何值，如果在控制台中打印 data.a，将返回一个 undefined。

其实这个问题很容易得到解决：

for (let key in data) {
  const dep = []
  let val = data[key] // 缓存字段原有的值
  Object.defineProperty(data, key, {
    set (newVal) {
      if (newVal === val) return // 如果值没有变什么都不做
      val = newVal // 使用新值替换旧值
      dep.forEach(fn => fn())
    },
    get () {
      dep.push(Target)
      return val  // 将该值返回
    }
  })
}

# 深度观测

以上，当数据 data 是嵌套的对象时，我们的程序只能检测到第一层对象的属性，比如数据对象如下：

const data = {
  a: {
    b: 1
  }
}

对于以上对象结构，我们的程序只能把 data.a 字段转换成响应式属性，而 data.a.b 依然不是响应式属性，但是这个问题还是比较容易解决的，只需要递归定义即可：

function walk (data) {
  for (let key in data) {
    const dep = []
    let val = data[key]
    // 如果 val 是对象，递归调用 walk 函数将其转为访问器属性
    const nativeString = Object.prototype.toString.call(val)
    if (nativeString === '[object Object]') {
      walk(val)
    }
    Object.defineProperty(data, key, {
      set (newVal) {
        if (newVal === val) return
        val = newVal
        dep.forEach(fn => fn())
      },
      get () {
        dep.push(Target)
        return val
      }
    })
  }
}

walk(data)

如上代码我们将定义访问器属性的逻辑放到了函数 walk 中，并增加了一段判断逻辑如果某个属性的值仍然是对象，则递归调用 walk 函数。这样我们就实现了深度定义访问器属性。

但是虽然经过上面的改造 data.a.b 已经是访问器属性了，但是如下代码依然不能正确执行：

$watch('a.b', () => {
  console.log('修改了字段 a.b')
})

这里，如果沿用之前的 $watch，实际上我们观测的应该是 data['a.b']，显然不对，我们我们需要对 $watch 做手脚了：

function $watch (exp, fn) {
  Target = fn
  let pathArr,
      obj = data
  // 检查 exp 中是否包含 .
  if (/\./.test(exp)) {
    // 将字符串转为数组，例：'a.b' => ['a', 'b']
    pathArr = exp.split('.')
    // 使用循环读取到 data.a.b
    pathArr.forEach(p => {
      obj = obj[p]
    })
    return
  }
  data[exp]
}

这里，解析了 exp，如果包含 . 就对其拆解，所以如果访问的路径是 a.b 那么转换后的数组就是 ['a', 'b']，然后使用一个循环从而读取到嵌套对象的属性值，不过需要注意的是读取到嵌套对象的属性值之后应该立即 return，不需要再执行后面的代码（因为此时已经触发了 get 函数）。
