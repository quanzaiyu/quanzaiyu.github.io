            # React 组件

            使用 React 组件需要注意以下几点

            • React 组件必须以大写字母开头
            • React 组件名不能为一个表达式
            • React 必须声明, import React from 'react';, 即使在代码中没有显式使用

            # 创建组件

            创建组件有两种方式:

            • 类定义组件
            • 函数定义组件
            import React from 'react';
class Hello extends Component {
  render () {
    return <div style={{color: this.props.color}}>Hello</div>
  }
}

function TextNotice (props) {
  return <div>{props.name}</div>
}

// or

// const TextNotice = props => (
//   <div>{props.name}</div>
// )

type Props = {};
export default class App extends React.Component<Props> {
  render() {
    return (
      <View style={styles.container}>
        <Hello color={'red'}/><TextNotice name='world'/>
      </View>
    );
  }
}

            在使用类定义组件时, 可以通过 this.props 接收指定的属性, 而函数定义的组件, 使用函数参数接收。

            # 组件状态(state)

            在 constructor 构造器中定义当前组件的 state, 通过 setState 改变组件状态值

            class App extends React.Component {
  constructor(props) {
    super(props);
    this.state = {
      title: null,
    };
  }

  render() {
    return (
      <button className="square" onClick={() => this.setState({title: 'X'})}>
        {this.state.title}
      </button>
    );
  }
}

            注意

            1. 在 constructor 中调用 super() 方法才能在继承父类的子类中正确获取到类型的 this 。
            2. 构造函数是唯一能够初始化 this.state 的地方。
            3. 不要直接更新状态, 比如 this.state.comment = 'Hello'; 是错误的写法, 而应该使用 this.setState({comment: 'Hello'});

            # 状态更新是异步的

            以下代码可能无法按预期更新:

            this.setState({
  counter: this.state.counter + this.props.increment,
});

            因为 this.props 和 this.state 可能是异步更新的，不应该依靠它们的值来计算下一个状态。

            要解决此问题, 可以使用以下的解决方案:

            this.setState((prevState, props) => ({
  counter: prevState.counter + props.increment
}));

            该函数将接收先前的状态作为第一个参数，将此次更新被应用时的props做为第二个参数

            # 组件传值(props)

            组件间通过 props 进行传值

            函数组件的props

            function App(props) {
  return (
    <div className="App">{props.title}</div>
  );
}

class Main extends React.Component {
  render() {
    <App title="hello react" />
  }
}

            类组件的props

            class App extends Component {
  constructor (props) {
    super(props)
  }
  render () {
    return <div className="App">{this.props.title}</div>
  }
}

            如果一个类组件定义了 constructor, 则必须调用 super(props) 才能正确使用 this.props 接收到传入的值, 若果没有 constructor, 可直接使用 this.props

            # 类型默认值

            使用 defaultProps 定义属性默认值

            class Hello extends Component {
  static defaultProps = {
    color: 'green'
  }
  render () {
    return <div style={{color: this.props.color}}>Hello</div>
  }
}

            # 类型检查

            使用 propTypes 定义类型检查规则

            import PropTypes from 'prop-types';
class Hello extends Component {
  static defaultProps = {
    color: 'green'
  }
  static propTypes = {
    // 接收string
    color: PropTypes.string.isRequired,
    // 接收array<object>
    todos: PropTypes.arrayOf(
      PropTypes.shape({
        id: PropTypes.number.isRequired,
        completed: PropTypes.bool.isRequired,
        text: PropTypes.string.isRequired
      })
    ),
    // 接收function
    onTodoClick: PropTypes.func
  }
  render () {
    return <div style={{color: this.props.color}}>Hello</div>
  }
}

            注意

            React.PropTypes 自 React v15.5 起已弃用。请使用 prop-types 库代替。

            检查的类型

            • any 任意类型
            • array
            • bool
            • func
            • number
            • object
            • string
            • node
            • element
            • symbol

            其他检查

            • isRequired 必须传入

            # 访问虚拟DOM(ref)

            通过 ref 可以访问虚拟DOM, 并对其进行操作。

            class Hello extends Component {
  constructor (props) {
    super(props)
    this.state = {
      color: 'red'
    }
  }
  getColor () {
    return this.state.color
  }
  render () {
    return <Text style={{color: this.state.color}}>Hello</Text>
  }
}

type Props = {};
export default class App extends Component<Props> {
  constructor (props) {
    super(props)
    this.state = {
      color: ''
    }
  }
  render() {
    return (
      <View style={styles.container}>
        <Text
          onPress={() => {
              this.setState({
                color: this.helloElement.getColor()
                // color: this.helloElement.state.color
              })
            }
          }
        >Click Me</Text>
        <Text>{this.state.color}</Text>
        {/* 错误示例: <Text>{this.helloElement.getColor()}</Text>*/}
        <Hello ref={el => this.helloElement = el}/>
      </View>
    );
  }
}

            注意

            看上述示例, 必须先通过 setState 拿到组件中的某些方法或属性, 而不能直接通过此 ref 直接调用其方法或属性, 因为当组件渲染时, 虚拟DOM还未转化为真实DOM。

            废弃: String 类型的 Refs

            老版本的react, 允许组件通过 String 类型创建 Refs, 比如在组件中声明 ref="helloElement", 可以通过 this.refs.helloElement 访问DOM节点, 现已废弃。

            参考: https://react.docschina.org/docs/refs-and-the-dom.html#%E6%97%A7%E7%89%88-api%EF%BC%9Astring-%E7%B1%BB%E5%9E%8B%E7%9A%84-refs

            # 条件渲染

            返回 null 或 false 的组件不会被渲染:

            function WarningBanner(props) {
  if (!props.warn) {
    return null;
  }

  return (
    <div className="warning">
      Warning!
    </div>
  );
}

class App extends React.Component {
  render() {
    return (
      <div>
        <div>
          <div>false: <WarningBanner warn={false}/></div>
          <div>true: <WarningBanner warn={true}/></div>
        </div>
      </div>
    );
  }
}

            # 点表示法

            如果一个模块包括多个组件, 可以使用点分割:

            import React from 'react';

const MyComponents = {
  DatePicker: function DatePicker(props) {
    return <div>Imagine a {props.color} datepicker here.</div>;
  }
}

function BlueDatePicker() {
  return <MyComponents.DatePicker color="blue" />;
}

            # 动态组件

            如果一个组件需要运行时确定, 则可以采用动态组件的方式:

            function Photo (props) {
  return <div>{props.story} PhotoStory</div>
}

function Video (props) {
  return <div>{props.story} VideoStory</div>
}

const components = {
  photo: Photo,
  video: Video
};

function Story(props) {
  // JSX 标签名可以为大写开头的变量。
  const SpecificStory = components[props.story];
  return <SpecificStory story={props.storyText}/>;
}

class App extends React.Component {
  render() {
    return (
      <Story story='video' storyText='long long ago'/>
    );
  }
}
