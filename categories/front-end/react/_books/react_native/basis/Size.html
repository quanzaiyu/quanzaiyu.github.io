# 尺寸

在 React Native 中尺寸是没有单位的，它代表了设备独立像素。

<View style={ {width:100,height:100,margin:40,backgroundColor:'gray'}}>
        <Text style={ {fontSize:16,margin:20}}>尺寸</Text>
</View>

上述代码，运行在Android上时，View的长和宽被解释成：100dp 100dp单位是dp，字体被解释成16sp 单位是sp，运行在iOS上时尺寸单位被解释称了pt，这些单位确保了布局在任何不同dpi的手机屏幕上显示不会发生改变；

# 屏幕适应

在屏幕尺寸与设计稿的比例上, 有以下公式:

设计稿元素宽度（px） / 设计稿总宽度（px） = 元素的宽度（dp） / 屏幕的总宽度（dp）

进而求得

元素的宽度（dp） = 设计稿元素宽度（px）* 屏幕的总宽度（dp） / 设计稿总宽度（px）

于是可以写出转换库:

import { Dimensions } from "react-native";

// 设备宽度，单位 dp
export const deviceWidthDp = Dimensions.get('window').width;
export const deviceHeightDp = Dimensions.get('window').height;

// 设计稿宽度，单位 px
const uiWidthPx = 750;

export function px(px) {
  return px * deviceWidthDp / uiWidthPx;
}

export function rem(rem) {
  return rem * deviceWidthDp / 10
}

export function vw(vw) {
  return vw * deviceWidthDp
}

export function vh(vh) {
  return vh * deviceHeightDp
}

使用的时候:

import px2dp from '../utils/px2dp';
import {StyleSheet} from 'react-native';

const style = StyleSheet.create({
  item: {
    width: px(700)
  },
  item: {
    width: vw(.5)
  },
  item: {
    width: rem(3)
  },
});

附:

  • PixelRatio.get() 获取设备像素比
  • Dimensions.get('window').width 获取设备的宽度
  • Dimensions.get('window').height 获取设备的高度
