# 多平台支持

通过 Platform 接口可以获取到当前平台信息，针对不同的平台设置特定的样式、组件结构等。

# 根据平台设置样式

Platform.OS 最常用的就是针对不同平台系统设置不同的样式：

import { Platform, StyleSheet } from "react-native";

const styles = StyleSheet.create({
  height: Platform.OS === "ios" ? 200 : 100
});

使用 Platform.select 可以更容易地设置样式：

const styles = StyleSheet.create({
  container: {
    flex: 1,
    ...Platform.select({
      ios: {
        backgroundColor: "red"
      },
      android: {
        backgroundColor: "blue"
      }
    })
  }
});

# 根据平台输出结构

比如一个Touchable组件，Android和iOS的支持程度不同，对此输出不同的结构：

<FlatList
  data={menus}
  renderItem={({item}) => (
    Platform.OS === "ios" ?
      <TouchableHighlight
        onPress={() => { navigation.navigate(item.path) }}
        activeOpacity={0.5}
        underlayColor='#ccc'
      >
        {renderItem(item)}
      </TouchableHighlight> :
      <TouchableNativeFeedback
        background={TouchableNativeFeedback.Ripple('#ccc', false)}
        onPress={() => { navigation.navigate(item.path) }}
      >
        {renderItem(item)}
      </TouchableNativeFeedback>
  )}
/>

# 获取平台版本

通过 Platform.Version 可以获取当前设备的平台系统版本

if (Platform.Version === 25) {
  console.log("Running on Nougat!");
}

# 特定平台扩展名

当不同平台的代码逻辑较为复杂时，最好是放到不同的文件里，这时候我们可以使用特定平台扩展名。React Native 会检测某个文件是否具有.ios.或是.android.的扩展名，然后根据当前运行的平台自动加载正确对应的文件。

比如你可以在项目中创建下面这样的组件：

BigButton.ios.js
BigButton.android.js

然后去掉平台扩展名直接引用：

import BigButton from './BigButton';

React Native 会根据运行平台的不同自动引入正确对应的组件。

如果你还希望在 web 端复用 React Native 的代码，那么还可以使用.native.js的扩展名。此时 iOS 和 Android 会使用BigButton.native.js文件，而 web 端会使用BigButton.js。
