            # CSS 引入网络字体

            # 基础语法

            使用 @font-face 可以引入网络字体，可以是本地路径，也可以是网络路径。

            @font-face{
    font-family: 'YaHei Consolas Hybrid';
    src : url('../fonts/yaheiconsolashybrid.ttf');
}

body{
    font-family: 'YaHei Consolas Hybrid';
    font-size: 16px;
}

            # 兼容写法

            不同浏览器字体的格式有差别：

            字体后缀和浏览器有关，如下所示

            • .TTF或.OTF，适用于Firefox 3.5、Safari、Opera
            • .EOT，适用于Internet Explorer 4.0+
            • .SVG，适用于Chrome、IPhone

            比如:

            @font-face {
    font-family: 'HansHandItalic';
    src: url('fonts/hanshand-webfont.eot');
    src: url('fonts/hanshand-webfont.eot?#iefix') format('embedded-opentype'),
         url('fonts/hanshand-webfont.woff') format('woff'),
         url('fonts/hanshand-webfont.ttf') format('truetype'),
         url('fonts/hanshand-webfont.svg#webfont34M5alKg') format('svg');
    font-weight: normal;
    font-style: normal;
}

            # 使用网络字体的问题

            由于中文字体文件过大，一般都3MB甚至10MB+，在浏览器加载的的时候很容易造成页面的卡顿，Chrome 浏览器甚至会在控制台输出：

            Slow network is detected. Fallback font will be used while loading

            对于字体的加载，有很多解决方案，一般来说分为"按需截取"（根据页面内容把字体中不需要的字型删除掉）与"整套嵌入"

            “按需截取”与“整套嵌入”方案相比：

            (1) “按需截取”生成的中文字体只有十几K至一百多K，而“整套嵌入”的中文字体一般3M至十几M，因此页面响应速度上“按需截取”狂甩“整套嵌入”好几条街。

            (2) “按需截取”是每一张页面都配有专属的字体文件（字体子集），即每打开一张页面就会加载与之对应的字体文件，而"整套嵌入“则是一次加载，整站通用。

            # 有字库解决方案

            针对“按需截取”的方案，有字库提供了很好的解决方案。

            分为两种模式：

            # CSS 模式：

            在线将需要加载的文字内容生成指定的css文件，通过引用的方式即可加载，比如：

            首先引入

            <link href='http://cdn.webfont.youziku.com/webfonts/nomal/122322/47629/5b7e784af629d91860518d9b.css' rel='stylesheet' type='text/css' />

            然后在需要设置的html中加入class

            class="css140b2f2a791ddd2"

            或者添加css属性：

            font-family:'zw2012sjt001140b2f2a791ddd2';

            # JS 模式

            在需要引入的地方加入以下JS即可：

            <script type="text/javascript" src="http://cdn.webfont.youziku.com/wwwroot/js/wf/youziku.api.min.js"></script>
<script type="text/javascript">
   $youziku.load("body", "4df80a1d3ac643a4929d891a4f4a4ce8", "winmantun21001");
   /*$youziku.load("#id1,.class1,h1", "4df80a1d3ac643a4929d891a4f4a4ce8", "winmantun21001");*/
   /*．．．*/
   $youziku.draw();
</script>

            注意，在生产环境中记得将自己的域名配置到有字库的白名单。

            # SFont解决方案

            除了有字库的解决方案之外，SFont 也是一个不错的解决方案，可以根据用户输入的文字生成指定的字体，并打包下载，也做到了“按需截取”。

            下载下来的字体有一个示例文件：

            可以使用class加载，也可以使用十六进制嵌入，为了方便，我喜欢使用十六进制嵌入：
