# 视图

首先搭建以下目录结构:

模板默认的目录位于当前模块的 View 目录，模板文件名为当前控制器命名的方法名。

也就是说，视图文件存放于 application/对应模块/view 目录下。

比如，在 controller/index.php 中，存在着一个名叫 index 的控制器，则在 view 目录下创建对应名称的，也就是 index.html 即可。

我们改写 index.php ，添加返回 view():

namespace app\index\controller;

class Index
{
  public function index()
  {
    return view();
  }
}

在 index.html 中，输入:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta http-equiv="X-UA-Compatible" content="ie=edge">
  <title>Document</title>
</head>
<body>
  <h1>Hello PHP !</h1>
</body>
</html>

此时，在浏览器中输入 http://www.my-tp5.com/ 即可看到效果。

TIP

如果全局设置中默认输出格式不为 html，可以在控制器方法中动态设置输出格式:

\think\Config::set('default_return_type','html');

以达到仅在指定方法中生效。

# View 对象

实际上，在控制器中可以使用 $this->view 返回一个 View 对象，即

$this->view === new \think\View()

调用其 display() 方法可以输出指定内容:

$view = new \think\View();
return $view->display('<h1>Hello xiaoyu</h1>');

在 TP5.1 中，View 类封装在 facade 中，可以使用静态代理

例如，以下代码

$display = '<h1>Hello xiaoyu</h1>';
return $this -> view -> display($display);

可以换为

$display = '<h1>Hello xiaoyu</h1>';
return View::display($display);

当然也可以省略为

$display = '<h1>Hello xiaoyu</h1>';
return $this -> display($display);

# 渲染当前视图

使用 fetch 方法返回当前控制器方法对应的视图。

public function index()
{
	return $this -> view -> fetch();
}

默认渲染的视图文件为 application/对应模块/view/当前控制器/当前方法同名的html文件

使用 fetch 可以指定调用的模板，并传值:

return $this->fetch('test', ['name' => 'xiaoyu']);

此时调用的就是 test.html 模板，并向模板传递一个 name 变量。

fetch 支持以下几种用法:

用法 描述
不带任何参数 自动定位当前操作的模板文件
[模块@][控制器/][操作] 常用写法，支持跨模块
完整的模板文件名 直接使用完整的模板文件名（包括模板后缀）

比如:

// 使用当前模块、当前控制器、当前方法同名的模板渲染
return $this->fetch();
// 使用当前模块、当前控制器下的 edit.html 模板渲染
return $this->fetch('edit');
// 使用当前模块 Member 控制器下的 read.html 模板渲染
return $this->fetch('member/read');
// 跨模块渲染模板: 使用 Admin 模块下 member 控制器下的 edit.html 模板渲染
return $this->fetch('admin@member/edit');

// 表示使用 当前模块/默认视图目录/test.html 进行渲染
return $this->fetch('/test');

// 指定一个完整的模板文件位置
// 要注意模板文件位置是相对于应用的入口文件，而不是模板目录。
return $this->fetch('./template/test.html');
// 也就是说如果入口文件为 /public/index.php，则调用的是 /public/template/test.html 模板文件。

// 全路径模板调用
return $view->fetch(APP_PATH.request()->module().'/view/public/header.html');

所有渲染操作还可以直接使用助手函数进行操作:

view('test',['name'=>'xiaoyu']);

# 向视图传值

在控制器中，可以使用 assign 方法向视图传值。

assign 接收一个 key-value 的字符串值，或者 一个数组，用于传递多个值。

public function index()
{
  $this -> assign('name', 'xiaoyu');
  $this -> assign([
    'age' => 22,
    'sex' => 'male'
  ]);
  return $this -> view -> fetch();
}

在模板中可以使用模板引擎调用传递的变量 :

  <h1>{$name}</h1>
  <p>{$age}</p>
  <p>{$sex}</p>

除了使用 assign 为模板传递变量外，fetch、display、view 的第二参数同样支持传递变量。

$this->fetch('index', [ 'name'  => 'Hello' ]);

$this->display('{$name}', [ 'name'  => 'Hello' ]);

view('index', [ 'name'  => 'Hello' ]);

# 渲染引擎

thinkphp 默认有自己的渲染引擎，视图类也提供了engine 方法对模板解析引擎进行初始化或者切换不同的模板引擎:

return $this->engine('php')->fetch();

也可使用第三方模板引擎，比如 angularjs 的模板，参见: https://www.kancloud.cn/shuai/php-angular
