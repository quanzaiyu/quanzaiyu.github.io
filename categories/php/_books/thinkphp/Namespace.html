# 命名空间

# 系统类库

在调用PHP内置的类库，或者第三方没有使用命名空间的类库的时候，记得在实例化类库的时候加上 \，例如：

// 错误的用法
$class = new stdClass();
$xml  =  new SimpleXmlElement($xmlstr);
// 正确的用法
$class = new \stdClass();
$xml  =  new \SimpleXmlElement($xmlstr);

# 根命名空间

系统会默认加载以下命名空间下的类库 :

名称 描述 类库目录
think 系统核心类库 thinkphp/library/think
traits 系统Trait类库 thinkphp/library/traits
app 应用类库 application

只有按照指定的规范命名才能实现自动加载，如果需要增加根命名空间，需要将这些类库放置于 extend 目录下，比如 :

extend/ext/Test.php

namespace ext;

class Test 
{
  public function sayHello()
  {
    echo 'hello';
  }
}

这样的话，就可以自动加载此命名空间下的类库，使用的时候 :

$test = new \ext\Test();
$test->sayHello();

如果你的扩展类库不希望放入系统默认的 extend 目录，就需要在应用配置文件中设置 root_namespace 注册根命名空间，例如下面注册了 ext 根命名空间到application/extend目录下面。

'root_namespace' => [
    'ext'  => '../application/extend/ext/'
]
