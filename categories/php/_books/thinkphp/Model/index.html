# 模型: 定义与设置

# 模型定义

在 TP 中，模型文件放置于 模块/model 目录下，一个模型对应一张数据表。

比如 数据库中有一个表名叫 user，那么只需要建立一张与此表名同名的模型即可:

基本结构:

application/index/model/User.php

namespace app\index\model;
class User extends \think\Model {}

# 模型参数

在模型中，可以自定义模型的各项参数。

属性 描述
name 模型名（默认为当前不含后缀的模型类名）
table 数据表名（默认自动获取）
pk 主键名（默认为id）
connection 数据库连接（默认读取数据库配置）
query 模型使用的查询类名称
field 模型对应数据表的字段列表（数组）

# 模型名

模型名默认自动获取，为模型类名。

如果指定，则连接到数据库中同名的 数据库表名前缀+指定的模型名 数据表，比如:

namespace app\index\model;

use think\Model;

class User extends model
{
  protected $name = 'test';
}

虽然模型类名为 User，但指定了模型名为 test，则连接到数据库中 数据库表名前缀+test，比如配置文件中指定数据库前缀: 'prefix' => 'api_'，则连接到数据库中 api_test 表。

# 数据表名

数据表名默认自动获取，为 数据库表名前缀+模型类名 数据表名可以指定为跟模型名不同。

若同时指定了模型名跟数据表名，则使用数据表名指定的数据表，需要指定不带前缀的数据表名，比如:

namespace app\index\model;

use think\Model;

class User extends model
{
  protected $name = 'test';
  protected $table = 'api_user';
}

上述例子，同时指定了 name 和 table，则连接到数据表 api_user。

# 主键

默认主键为 id，如果表中没有使用 id 作为主键名，需要在模型中设置属性

注意

5.1中模型不会自动获取主键名称，必须设置pk属性。

protected $pk = 'uid';

# 数据库连接

甚至可以改变数据库连接，配置一个数组，更改整个数据库的连接

protected $connection = [
  'type'        => 'mysql',
  'hostname'    => '127.0.0.1',
  'database'    => 'test',
  'username'    => 'root',
  'password'    => '',
  'charset'     => 'utf8',
  'prefix'      => 'api_',
];

# 模型操作

提示

在模型中除了可以调用数据库类的方法之外，也可以定义自己的方法，所以也可以把模型看成是数据库的增强版。

使用数据库操作

// 注意: 使用 name 指定的数据表不需要带表前缀，使用 table 指定的数据表需要使用完整的表名
\think\Db::name('user')->where('id','<',10)->select();
\think\Db::table('api_user')->where('id','<',10)->select();

使用模型的静态操作

// 假设模型的命名空间为 \app\index\model，模型类名为 User
\app\index\model\User::where('id','<',10)->select();

创建模型的实例进行操作

$db = new \app\test\model\User();
$db->where('id', '<', 10)->select();

使用 Loader 类实例化（单例）

// 会查找同一模块下的同名模型
$user = Loader::model('User');
$user->where('id', '<', 10)->select();

使用助手函数

// 会查找同一模块下的同名模型
model('User')->where('id', '<', 10)->select();

助手函数同样可以实例化模型对象:

$user = model('User);

# 模型初始化: initialize

模型通过重写 Model 的 initialize 方法进行初始化。

namespace app\index\model;

use think\Model;

class Index extends Model
{
  protected function initialize()
  {
    // 需要调用`Model`的`initialize`方法
    parent::initialize();
    // TODO: 自定义的初始化
  }
}

# init

使用静态 init 方法，需要注意的是 init 只在第一次实例化的时候执行。

namespace app\index\model;

use think\Model;

class User extends Model
{
  // 模型初始化
  protected static function init()
  {
    // TODO: 初始化内容
  }
}

模型初始化方法通常用于注册模型的事件操作。

提示

init 必须是静态方法，并且只在第一次实例化的时候执行。
