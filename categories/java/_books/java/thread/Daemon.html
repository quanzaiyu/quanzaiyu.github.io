# 守护线程

线程可以是守护线程或用户线程。

守护线程是服务提供者线程。

当JVM检测到应用程序中的所有线程都只是守护线程时，它将退出应用程序。

我们可以通过使用setDaemon()方法通过传递true作为参数，使线程成为一个守护线程。

我们必须在启动线程之前调用一个线程的setDaemon()方法。否则，一个java.lang。抛出IllegalThreadStateException。

我们可以使用isDaemon()方法来检查线程是否是守护线程。

创建线程时，其守护程序属性与创建线程的线程相同。

示例:

在没有设置守护线程的情况下:

public class Main {
    public static void main(String[] args) {
        Thread t = new Thread(Main::print);
        t.setDaemon(false);
        t.start();
        System.out.println("Exiting main  method");
    }
    private static void print() {
        int counter = 1;
        while (true) {
            try {
                System.out.println("Counter:" + counter++);
                Thread.sleep(2000); // sleep for 2 seconds
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

运行结果:

Exiting main  method
Counter:1
Counter:2
...

解释:

由于这个程序有一个非守护线程，JVM将继续运行应用程序，即使在main()方法完成后。

因此必须强制停止此应用程序，因为线程在无限循环中运行。

以下代码创建一个线程并将线程设置为守护线程:

public class Main {
    public static void main(String[] args) {
        Thread t = new Thread(Main::print);
        t.setDaemon(true);
        t.start();
        System.out.println("Exiting main method");
    }

    private static void print() {
        int counter = 1;
        while (true) {
            try {
                System.out.println("Counter:" + counter++);
                Thread.sleep(2000); // sleep for 2 seconds
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

运行结果:

Exiting main method
Counter:1

解释:

由于设置了守护线程, 在主线程运行完成后, 守护线程也将停止运行。
