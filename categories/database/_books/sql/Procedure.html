# 存储过程

存储过程是一组可编程的函数，是为了完成特定功能的SQL语句集，经编译创建并保存在数据库中，用户可通过指定存储过程的名字并给定参数(需要时)来调用执行。

为什么要用存储过程？

  • ① 将重复性很高的一些操作，封装到一个存储过程中，简化了对这些SQL的调用
  • ② 批量处理：SQL+循环，减少流量，也就是“跑批”
  • ③ 统一接口，确保数据的安全

相对于oracle数据库来说，MySQL的存储过程相对功能较弱，使用较少。

  • 存储过程就是具有名字的一段代码，用来完成一个特定的功能。
  • 创建的存储过程保存在数据库的数据字典中。

# 定义存储过程

CREATE PROCEDURE `NewProc`()
BEGIN
  #Routine body goes here...

END;

如果使用 Navicat 创建，默认会有以下模板：

CREATE DEFINER = CURRENT_USER PROCEDURE `NewProc`()
BEGIN
  #Routine body goes here...

END;

在 body 部分添加需要处理的语句

比如：

CREATE DEFINER=`root`@`localhost` PROCEDURE `NewProc`()
BEGIN
	SELECT * from t_user;
END

# 调用存储过程

使用 call 可以调用存储过程。

call NewProc();

# 存储过程体

  • 存储过程体包含了在过程调用时必须执行的语句，例如：dml、ddl语句，if-then-else 和 while-do 语句、声明变量的 declare 语句等
  • 过程体格式：以 begin 开始，以 end 结束 (可嵌套)
BEGIN
  BEGIN
    BEGIN
      statements;
    END
  END
END

注意

每个嵌套块及其中的每条语句，必须以分号结束，表示过程体结束的begin-end块(又叫做复合语句compound statement)，则不需要分号。

# 为语句块贴标签

[begin_label:] BEGIN
　　[statement_list]
END [end_label]

比如：

label1: BEGIN
　　label2: BEGIN
　　　　label3: BEGIN
　　　　　　statements;
　　　　END label3 ;
　　END label2;
END label1

标签有两个作用：

  • 增强代码的可读性
  • 在某些语句(例如: leave 和 iterate 语句)，需要用到标签

# 带参数的存储过程

存储过程可以有0个或多个参数，用于存储过程的定义。有3种参数类型：

  • IN 输入参数：表示调用者向过程传入值（传入值可以是字面量或变量）
  • OUT 输出参数：表示过程向调用者传出值(可以返回多个值)（传出值只能是变量）
  • INOUT 输入输出参数：既表示调用者向过程传入值，又表示过程向调用者传出值（值只能是变量）

注意

① 如果过程没有参数，也必须在过程名后面写上小括号

例：CREATE PROCEDURE sp_name ([proc_parameter[,...]]) ……

② 确保参数的名字不等于列的名字，否则在过程体中，参数名被当做列名来处理

# In 输入参数

一个简单的例子：

CREATE DEFINER=`root`@`localhost` PROCEDURE `NewProc`(IN id INTEGER)
BEGIN
	SELECT * from t_user where user_id=id;
END

这里，传入一个参数 id，在调用的时候可以传入：

call NewProc(1);

另一个例子：

CREATE DEFINER=`root`@`localhost` PROCEDURE `NewProc`(In out_num INTEGER)
BEGIN
  select out_num;
	set out_num = 1024;
  select out_num;
END

调用的时候：

set @p_out = 2;
call NewProc(@p_out);
select @p_out;

输出：

|---------|
| out_num |
|---------|
| 2       |
|---------|


|---------|
| out_num |
|---------|
| 1024    |
|---------|


|--------|
| @p_out |
|--------|
| 2      |
|--------|

# Out 输出参数

一个简单的例子：

CREATE DEFINER=`root`@`localhost` PROCEDURE `NewProc`(OUT out_num INTEGER)
BEGIN
  select out_num;
	set out_num = 1024;
  select out_num;
END

调用的时候：

set @p_out = 2;
call NewProc(@p_out);
select @p_out;

输出：

|---------|
| out_num |
|---------|
| (NULL)  |
|---------|


|---------|
| out_num |
|---------|
| 1024    |
|---------|


|--------|
| @p_out |
|--------|
| 1024   |
|--------|

# Inout 输入输出参数

调用了inout_param存储过程，接受了输入的参数，也输出参数，改变了变量。

CREATE DEFINER=`root`@`localhost` PROCEDURE `NewProc`(INOUT out_num INTEGER)
BEGIN
  select out_num;
	set out_num = 1024;
  select out_num;
END

调用的时候：

set @p_out = 2;
call NewProc(@p_out);
select @p_out;

输出：

|---------|
| out_num |
|---------|
| 2       |
|---------|


|---------|
| out_num |
|---------|
| 1024    |
|---------|


|--------|
| @p_out |
|--------|
| 1024   |
|--------|

# 参数的使用建议

  1. 输入值使用 in 参数；
  2. 返回值使用 out 参数；
  3. inout 参数就尽量的少用。
