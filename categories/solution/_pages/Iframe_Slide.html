        # 关于 iOS 手机 iframe 滑动不流畅的问题

        最近有个需求，要在页面中嵌入 iframe，加载用户协议。由于用户协议是直接用 MS Word 编辑的，生成的 html，而在 Vue 中直接将那个静态 HTML 代码装进去似乎不是太好。因此，采用 iframe 的方式嵌入页面。

        # 问题重现

        在 PC 上，使用 Chrome 浏览器模拟手机进行测试，没有任何问题，在安卓手机上也没任何问题，结果，在iPhone上出问题了：滑动的时候容器在上下滚动，而内容却没动。

        实际上，在样式上也是非常正常的用法，还在 iframe 中使用了 -webkit-overflow-scrolling: touch; 以兼容iOS系统。然而，就是木有效果。。。

        # 问题解决

        网上查了下资料，索性很容易就找到了解决方案，-webkit-overflow-scrolling: touch; 放到容器中而不是 iframe 中.

        比如有如下 HTML:

        <div class="scroll-wrapper">
 <iframe src=""></iframe>
</div>

        .scroll-wrapper {
  -webkit-overflow-scrolling: touch;
  overflow-y: scroll;
  /* 提示: 请在此处加上需要设置的大小(dimensions)或位置(positioning)信息! */
}
.scroll-wrapper iframe {
  /* 你自己指定的样式 */
}

        应用到我的项目中：

        <template lang="pug">
div
  //- 用户协议
  .dyt-protocol
    .iframe-wrap
      iframe(:src="src")
    .dyt-btn-group
      .btn.commit(@click='goBack') 关闭
</template>

        .dyt-protocol {
  position: absolute;
  right: 0;
  bottom: 0;
  left: 0;
  top: 0;
  .iframe-wrap {
    width: 100%;
    height: 88%;
    -webkit-overflow-scrolling: touch;
    overflow-y: scroll;
    iframe {
      width: 100%;
      height: 100%;
      border: none;
    }
  }
  .dyt-btn-group {
    width: 100%;
    height: 12%;
  }
}
