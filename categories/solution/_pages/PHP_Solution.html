    # PHP 常见问题汇总

    # 解决 PHP CURL 证书错误

    在使用 CURL 调用接口的时候，如果直接使用通常不会直接返回我们想要的结果，需要配置证书才行。

    # 解决步骤

    1. 下载 cacert.pem 保存到本地。
    2. 配置 php.ini，指定以下字段为 cacert.pem 存放路径
    curl.cainfo ="D:\phpStudy\cacert.pem"

    # Nginx 服务器中 PHP 调用本地接口失败的解决方案

    # 问题现象

    当在同一个 PHP 项目中访问同一个项目中的接口的时候，会出现访问超时、加载不出数据甚至卡死的问题。

    # 问题原因

    nginx 中，看 PHP 文件块 fastcig-pass 的设置值(127.0.0.1:9000)。设置都是以 keepalive 方式请求，接收到 PHP 文件时，交于后端过程 PHPCGI 解析处理(127.0.0.1:9000)，等待响应。

    而在本地文件以 CURL 请求本地环境中 PHP 文件时，之前的 PHP 还在等待 CURL 后的结果，这时 9000 端口已经被占用。导致 CURL 一直在处于等待状态。不设置 timeout 超时，程序就会卡死。

    # 解决方案

    第一步，使用负载均衡的方式为 php-cgi 设置多个端口，比如:

    upstream backend {
  server 127.0.0.1:9000;
  server 127.0.0.1:9001;
}

server {
  listen 80;
  server_name  www.dyt_api.com ;
  root   "D:/projects/dyt_api/public";
  location / {
    index  index.html index.htm index.php;
    #autoindex  on;
    if (!-e $request_filename) {
      rewrite  ^/(.*)$  /index.php/$1  last;
      break;
    }
  }
  location ~ \.php(.*)$ {
    fastcgi_pass   backend;
    fastcgi_index  index.php;
    fastcgi_split_path_info  ^((?U).+\.php)(/?.+)$;
    fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;
    fastcgi_param  PATH_INFO  $fastcgi_path_info;
    fastcgi_param  PATH_TRANSLATED  $document_root$fastcgi_path_info;
    include        fastcgi_params;
  }
}

    第二步，重启 nginx，并新开启一个 phpcgi 进程监听不同端口，比如:

    D:\phpStudy\php70n\php-cgi.exe -b 127.0.0.1:9001 -c D:\phpStudy\php70n\php.ini

    使用 netstat -no 命令查看端口是否工作正常。
