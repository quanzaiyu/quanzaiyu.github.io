  • 关于 ThinkPHP 使用数据库分库的方案

    • 最初尝试：只改用数据库名
      • 继续尝试：使用完整的数据库连接
        • 继续尝试：构造函数中改变连接参数
          • 最终解决：配置文件

          # 关于 ThinkPHP 使用数据库分库的方案

          最近着手一个新项目，由于数据库主库太庞大，想要重新创建一个数据库。

          最开始，觉得很简单，只需要换掉数据库名称即可，而其他的连接数据，比如数据库连接名、用户名和密码都不需要改变。

          首先，数据库配置文件 'database.php' 为：

          <?php
return [
  'database'  => 'dyt_db',
  'hostname'  => 'localhost',
  'type'      => 'mysql',
  'username'  => 'root',
  'password'  => 'root',
  'prefix'    => '',
  'hostport'  => '3306'
];

          # 最初尝试：只改用数据库名

          class InsuranceProvider extends Model
{
  protected $pk = 'provider_id';
  protected $table = 'api_insurance_provider';
  protected $connection = [
    'database'  => 'dyt_insurance'
  ];
}

          结果，抛出一个错误：throw new \InvalidArgumentException('Underfined db type');。

          # 继续尝试：使用完整的数据库连接

          好吧，看样子要指定 $connection 就必须将所有的参数都配置上，但是其他参数我需要拿默认配置的参数，只需要改变 database 这一个参数。由于线上服务器肯定配置的不是本地主机，因此我不能把其他参数写死，而是拿配置文件中的参数：

          class InsuranceProvider extends Model
{
  protected $pk = 'provider_id';
  protected $table = 'api_insurance_provider';
  protected $connection = [
    'database'  => 'dyt_insurance',
    'hostname'  => config('database.hostname'),
    'type'      => config('database.type'),
    'username'  => config('database.username'),
    'password'  => config('database.password'),
    'prefix'    => config('database.prefix'),
    'hostport'  => config('database.hostport')
  ];
}

          结果，报了一个更严重的错：Fatal error: Constant expression contains invalid operations。

          # 继续尝试：构造函数中改变连接参数

          看样子不能在数组中使用方法，接着尝试，那就在构造函数中试试：

          class InsuranceProvider extends Model
{
  public function __construct()
  {
    //  $this->connection = parent::getConnection();
    $type = config('database.type');
    $hostname = config('database.hostname');
    $username = config('database.username');
    $password = config('database.password');
    $hostport = config('database.hostport');
    $prefix = config('database.prefix');
    $this->table = 'api_insurance_provider';
    $this->pk = 'provider_id';
    $this->connection = [
      'database'  => 'dyt_insurance',
      'hostname'  => $hostname,
      'type'      => $type,
      'username'  => $username,
      'password'  => $password,
      'prefix'    => $prefix,
      'hostport'  => $hostport
    ];
  }
}

          然后在控制器中使用，咦，通了！兴奋之余，将使用到 dyt_insurance 库的模型全部用这种方式配置，完美。

          比如另一个模型：

          <?php
public function __construct()
{
//  $this->connection = parent::getConnection();
  $type = config('database.type');
  $hostname = config('database.hostname');
  $username = config('database.username');
  $password = config('database.password');
  $hostport = config('database.hostport');
  $prefix = config('database.prefix');
  $this->table = 'api_insurance_provider';
  $this->pk = 'provider_id';
  $this->connection = [
    'database'  => 'dyt_insurance',
    'hostname'  => $hostname,
    'type'      => $type,
    'username'  => $username,
    'password'  => $password,
    'prefix'    => $prefix,
    'hostport'  => $hostport
  ];
}

          然而，但我在同一个控制器的同一个方法中实例化不同的模型，会出现问题：

          $b = new InsuranceProvider();
dump($b::all());
$a = new InsuranceServices();
dump($a::all());

          这时，出现一个问题，$a 返回的数据居然跟 $b 一样！！！如果颠倒 $a 和 $b 的位置，$b 返回的数据跟 $a 一样！！！

          神奇的一幕。最终各种调试，也未发现问题所在。

          # 最终解决：配置文件

          最终只能通过新建额外的配置文件解决了，在 application/extra 下面新建了一个数据库配置文件：db_config_insurance.php，输入：

          <?php
return [
    'type'            => 'mysql',
    'hostname'        => 'localhost',
    'database'        => 'dyt_insurance',
    'username'        => 'root',
    'password'        => 'root',
    'hostport'        => '3306',
    'prefix'          => 'api_'
];

          然后在模型中配置：

          class InsuranceProvider extends Model
{
  protected $pk = 'provider_id';
  protected $table = 'api_insurance_provider';
  protected $connection = 'db_config_insurance';
}

          创建了多个模型，控制器中照常调用，问题终于解决了。

          接下来要解决的是远程服务器数据库连接跟本地不一致的问题，第一次 git 提交，将 db_config_insurance.php 修改为服务器端的数据库连接，第二次提交，在 .gitignore 中将 application/extra/db_config_insurance.php 加入，本地的 db_config_insurance.php 改为本地数据库连接，之后提交都不影响此文件，问题终于解决了。

          总结：其实我已开始就是知道要用额外的配置文件解决，但又不想通过 gitignore 的方式添加，由此做了一系列的探索。
