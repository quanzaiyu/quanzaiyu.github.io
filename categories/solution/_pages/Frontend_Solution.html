                    # 前端问题汇总及解决方案

                    # input:file 在 Android 中 onchange 不生效的处理

                    如果在 accept 指定具体的文件类型, 在 Android 版的微信中 onchange 事件可能触发不了

                    <input type="file" accept="image/gif, image/jpeg"  onchange="fileChange" />

                    需要改成下面形式

                    <input type="file" accept="image/*"  onchange="fileChange" />

                    同理, 如果是 audio 和 video, 也是同样的解决方案

                    # vue-cli3 出现 Invalid Host header 的解决方案

                    问题详情: 用vue-cli3搭建了一个项目，结果一运行不停的报错，报错信息如下：

                    Invalid Host/Origin header

                    产生原因: 新版的webpack-dev-server增加了安全验证，默认检查hostname，如果hostname不是配置内的(比如改了hosts后访问)，将中断访问。

                    解决方案: 在根目录下创建文件vue.config.js，然后填入如下内容

                    module.exports = {
    devServer: {
        disableHostCheck: true,
    }
}

                    重启服务即可

                    参考:

                    • vue-cli3出现Invalid Host header的解决方案
                    • Invalid Host header 服务器域名访问出现的问题

                    # Vue 报 runtime-only 错误

                    错误详情:

                    [Vue warn]: You are using the runtime-only build of Vue where the template compiler is not available. Either pre-compile the templates into render functions, or use the compiler-included build.

                    在 webpack 中配置:

                    configureWebpack: {
  resolve: {
    alias: {
      'vue$': 'vue/dist/vue.esm.js'
    }
  }
}

                    参考: You are using the runtime-only build of Vue where the template compiler is not available.

                    # 使用fetch上传文件失败的解决方案

                    使用 fetch(url,formData) ，指定 content-type:multipart/form-data，但是后台怎么也拿不到文件，用ajax可以正常上传。

                    比较了一下两个请求的区别在于，ajax请求中 content-type:multipart/form-data; boundary=----WebKitFormBoundaryAnZIsvCKxDSH7vQ0

                    fetch请求中 content-type:multipart/form-data 后面没有 boundary=----WebKitFormBoundaryAnZIsvCKxDSH7vQ0

                    解决方案就是删除content-type

                    参考: 使用fetch上传文件失败的解决方案

                    # Less 设置 '/' 时的问题

                    在使用 Less 的时候，如果设置的值里面包含 /，是不会进行转义的，Less 解析器会将之当成除法运算，比如:

                    grid-area: 2 / 2 / 5 / 4;

                    这输出的结果为:

                    grid-area: 0.05;

                    解决方案为: 使用 e('/) 进行转义

                    grid-area: 2 e('/') 2 e('/') 5 e('/') 4;

                    输出为:

                    grid-area: 2 / 2 / 5 / 4;

                    参考: 使用Less时，CSS值中的斜杠（/）

                    # iOS 系统时间解析

                    iOS 系统在使用时间对象的时候一定得注意了，如果使用:

                    let date = '2018-06-08 06:30:00'
new Date(date)

                    这种形式创建一个事件对象，iOS是不成功的，需要将 - 转换为 /

                    let date = '2018-06-08 06:30:00'
new Date(date.replace(/-/g, '/'))

                    同样的，在iOS中，将Date对象转化为 - 格式的字符串也有问题：

                    new Date().toLocaleDateString().replace(/\//g, '-')

                    我在iOS手机上打印出来为 2018年1月1日 这种格式。

                    懒得折腾了，最后引用了 moment 库解决：

                    moment(new Date()).format('YYYY-MM-DD')

                    这个坑不大，但很容易掉沟里，我着过好几次。

                    # iOS 系统滑动的问题

                    如果内容溢出容器，在iOS系统的手机上滑动会非常吃力，一顿一顿的，还不时连同绝对定位的元素一起开始飘，这时，只需要在容器中加入以下代码即可解决：

                    .container {
	-webkit-overflow-scrolling:touch;
}

                    注意是加在有宽高的容器元素中，而不是内容元素中：

                    .container {
	width: 100px;
	height: 100px;
	overflow-x: hidden;
	overflow-y: scroll;
	-webkit-overflow-scrolling:touch;
	.content {
  	width: 100px;
  	height: 500px;
  }
}

                    或者最终渲染出来仍然具有宽高但是却不是显式生命的容器中：

                    .container {
	top: 0;
	bottom: 0;
	left: 0;
	bottom: 0;
	overflow-x: hidden;
	overflow-y: scroll;
	-webkit-overflow-scrolling:touch;
	.content {
		width: 100%;
		height: 500%;
	}
}

                    简单说说 -webkit-overflow-scrolling 属性，MDN 是这样说的：

                    `-webkit-overflow-scrolling` 属性控制元素在移动设备上是否使用滚动回弹效果.

- auto: 使用普通滚动, 当手指从触摸屏上移开，滚动会立即停止。
- touch: 使用具有回弹效果的滚动, 当手指从触摸屏上移开，内容会继续保持一段时间的滚动效果。继续滚动的速度和持续的时间和滚动手势的强烈程度成正比。同时也会创建一个新的堆栈上下文。

                    参考：

                    • 深入研究-webkit-overflow-scrolling:touch及ios滚动
                    • preventoverscrolljs - npm

                    # img标签src为空

                    在Vue中使用的时候, 控制台并没有任何报错, 但是我们使用了bug监控工具, 用户使用时只要抛出异常就会往数据库写数据, 密密麻麻的一大堆都是img标签src为空的问题, 虽然不影响正常使用, 但是看着这一堆的报错确恶心

                    由于数据是异步获取, 所以在初始化的是否数据并没有获取到, 渲染为 <img src="" />, 可以通过 v-if 判断数据是否获取到, 比如 <img :src="img.url" v-if='img.url'/>

                    参考:

                    • img标签src为空的陷阱-Web

                    # audio 语音播放在手机端使用注意

                    在手机端需要先播放 player.play()，否则监听不到 canplay

                    但是监听到了就要马上 player.pause() 停止播放；防止项目需求上没有需要播放的也播放了
