Git 简易教程

配置

使用 git 之前，需要做一些简单配置，包括用户名和邮箱

git config --global user.email "you@example.com"
git config --global user.name "Your Name"
1
2

如

root@ubuntu:~/temp/git-local# git config --global user.email "731734107@qq.com"
root@ubuntu:~/temp/git-local# git config --global user.name "xiaoyu"
1
2

创建新仓库

git init
1

检出仓库

执行如下命令以创建一个本地仓库的克隆版本：

git clone /path/to/repository
1

如果是远端服务器上的仓库：

使用 SSH

git clone username@host:/path/to/repository
1

使用 HTTPS

git clone https://github.com/path/to/repository.git
1

根据以上知识创建一个 git 仓库，并在客户端完成克隆 :

添加一个远程仓库

如果在本地已经初始化了一个本地仓库，需要添加远程仓库，可以使用

git remote add <local-name> <server>
1

比如 :

root@ubuntu:~/temp# mkdir git-local && cd git-local && git init
Initialized empty Git repository in /root/temp/git-local/.git/
root@ubuntu:~/temp/git-local# git remote add origin /root/git-test/
1
2
3

此时在本地仓库命名了一个叫 origin 的远程仓库，之后只需要使用此名称就可将变更推送到指定远程仓库。

查看 git 配置文件可以看到添加成功了 :

root@ubuntu:~/temp/git-local/.git# cd .git && cat config 
[core]
	repositoryformatversion = 0
	filemode = true
	bare = false
	logallrefupdates = true
[remote "origin"]
	url = /root/git-test/
	fetch = +refs/heads/*:refs/remotes/origin/*

添加与提交

把计划改动的文件添加到缓存区，使用如下命令：

git add <filename>
git add *
git add .
1
2
3

这是 git 基本工作流程的第一步；使用如下命令以实际提交改动：

git commit -m "代码提交信息"
1

现在，改动的文件已经提交到了 HEAD，但是还没同步到远端仓库。

测试 :

root@ubuntu:~/temp/git-local# vim test.txt
root@ubuntu:~/temp/git-local# cat test.txt
learn git !
root@ubuntu:~/temp/git-local# git add .
root@ubuntu:~/temp/git-local# git commit -m 'first commit'
[master (root-commit) fe6366d] first commit
 1 file changed, 1 insertion(+)
 create mode 100644 test.txt

更新与合并

要更新本地仓库至最新改动，执行：

git pull
git pull origin master
1
2

git pull 实际上是一个合并的命令，包括 获取（fetch）并 合并（merge） 远端的改动。

要合并其他分支到你的当前分支（例如 master），执行：

git merge <branch>
1

两种情况下，git 都会尝试去自动合并改动。但自动合并并非次次都能成功，并可能导致 冲突（conflicts）。

这时候就需要你修改这些文件来人为合并这些 冲突（conflicts） 。改完之后，你需要执行如下命令以将它们标记为合并成功：

git add <filename>
1

在合并改动之前，也可以使用如下命令查看：

git diff <source_branch> <target_branch>
1

测试 :

root@ubuntu:~/git-test# vim a
root@ubuntu:~/git-test# git add *
root@ubuntu:~/git-test# git commit -am 0

推送改动

改动已经在本地仓库的 HEAD 中了。执行如下命令以将这些改动提交到远端仓库：

git push origin master
1

origin 为绑定的远程仓库名称，master 为需要推送到的分支，可以把 master 换成想要推送的任何分支。

如果首次提交失败，提示 :

By default, updating the current branch in a non-bare repository is denied, because it will make the index and work tree inconsistent with what you pushed, and will require 'git reset --hard' to match the work tree to HEAD.
1

可以在远程仓库设置 git config receive.denyCurrentBranch ignore 忽略之

root@ubuntu:~/git-test# git config receive.denyCurrentBranch ignore
root@ubuntu:~/git-test# cd ../temp/git-local/
root@ubuntu:~/temp/git-local# git push origin master

分支

分支是用来将特性开发绝缘开来的。在你创建仓库的时候，master 是“默认的”。在其他分支上进行开发，完成后再将它们合并到主分支上。

创建一个叫做“feature_x”的分支，并切换过去：

git checkout -b feature_x
1

切换回主分支：

git checkout master
1

再把新建的分支删掉：

git branch -d feature_x
1

除非你将分支推送到远端仓库，不然该分支就是 不为他人所见的：

git push origin <branch>
1

标签

在软件发布时创建标签，是被推荐的。这是个旧有概念，在 SVN 中也有。可以执行如下命令以创建一个叫做 1.0.0 的标签：

git tag 1.0.0 1b2e1d63ff
1

1b2e1d63ff 是你想要标记的提交 ID 的前 10 位字符。使用如下命令获取提交 ID：

git log
1

你也可以用该提交 ID 的少一些的前几位，只要它是唯一的。

替换本地改动

假如你做错事，你可以使用如下命令替换掉本地改动：

git checkout -- <filename>
1

此命令会使用 HEAD 中的最新内容替换掉你的工作目录中的文件。已添加到缓存区的改动，以及新文件，都不受影响。

假如你想要丢弃你所有的本地改动与提交，可以到服务器上获取最新的版本并将你本地主分支指向到它：

git fetch origin
git reset --hard origin/master
1
