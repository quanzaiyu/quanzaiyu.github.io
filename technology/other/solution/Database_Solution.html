      # 数据库问题汇总及解决方案

      # [Err] 1215 - Cannot add foreign key constraint

      今天在 MySQL 数据表中添加外键的时候，遇到以下报错：

      [Err] 1215 - Cannot add foreign key constraint
      1

      导致外键无法添加。

      外键定义添加的条件：

      • 外键对应的字段数据类型保持一致
      • 所有 tables 必须是 InnoDB 型，它们不能是临时表.因为在 MySQL 中只有 InnoDB 类型的表才支持外键（两张表的存储引擎一致）。
      • 设置外键时“删除时”设置为“SET NULL”

      最后又复查了一下两张表的定义，发现一个是 int 类型，另一个是 int UNSIGNED，不满足条件一，将其数据类型改为一致，即可成功添加。

      参考：

      数据库添加外键错误：[Err] 1215 - Cannot add foreign key constraint 的解决

      # [Err] 1050 Table 'xxx' already exists

      错误描述：在修改一张表的某个字段类型是突然报了那么一个错误。

      解决方案：

      删除表再重新创建对应的表。
