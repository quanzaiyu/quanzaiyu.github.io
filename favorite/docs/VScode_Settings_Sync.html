              # 使用 Settings Sync 同步 VSCode 设置到 GitHub

              一开始不知道怎么备份 vscode 的配置，将所有的插件列表、配置文件写到我的博客里面，每次在新的机子里面装 vscode 的时候又照着博客一个一个安装，折腾得要死。虽然大部分人可能都不会遇到重装系统的麻烦，但像我这种经常换系统的单纯码农就明显感觉得到这是有多麻烦，特别是最近着手着公司的 Mac 电脑，又得赶项目又得配环境，别提多难受。

              今天要说的就是这个插件 Settings Sync，一个可以在 github 上面通过 gist 仓来同步用户的 vscode 的配置，包括插件，settings 等信息的好插件。

              # 创建 Gist

              打开 github 的创建 gist 的页面: https://github.com/settings/tokens，然后在 github 勾选 gist 选项来存储设置

              保存后会生成一个 key，注意这个 key 只会出现一次，现在一定要将它保存下来

              # 上传配置

              1. 安装 Settings Sync 插件并重启 vscode
              1. 重启后按快捷键 alt + shift + u, 或键入 ctrl + shift + p，输入 sync，点击 Sync: Upload / Update Settings, 输入刚才记下的 gist Id

              不出问题的话它就开始对你本机的配置进行一个扫描上传了。至此上传工作完成。

              # 同步配置

              接下来我们要在另一台机子上同步设置。同样的先安装 Settings Sync 插件，并重新加载。

              然后按快捷键 alt + shift + d，就应该会弹出一个输入框，请在这里输入之前保存下来的 key(GIST ID)，回车后将会自动下载之前上传的配置。

              那么下载完成后，你这台电脑修改了相关配置再次上传就好了。

              # 修改设置

              如果在输入 gist id 写错了，读取不到的情况下，就需要重置设置，按 ctrl(cmd)+shift+P，输入 sync，这里就有重置选项

              # Invalid / Expired GitHub Token

              同步时提示:

              Sync : Invalid / Expired GitHub Token. Please generate new token with scopes mentioned in readme. Exception Logged in Console.

              解决方法:

              1. 重新生成token, 到 https://github.com/settings/tokens 中找到需要修改的gist, 点击 Regenerate token, 拿到重新生成的token
              2. 在VSCode中修改, 点击命令面板 或 Ctrl+Shift+P 输入 sync: 找到 Sync: Advanced Options 回车, 选中 Edit Extension Local Settings
              3. 修改配置文件中的 token 属性就可以了
