# 保存时自动格式化成ESLint风格

使用 VSCode 自带的 Ctrl + Shift + F 并不能把代码保存成 ESLint 风格, 可以借助 VSCode 的 ESLint 插件做到这一点。

打开用户配置 文件 -> 首选项 -> 设置:

{
  "eslint.autoFixOnSave": true,
  "eslint.alwaysShowStatus": true,
  "eslint.validate": [
    "javascript",
    "javascriptreact",
    {
      "language": "html",
      "autoFix": true
    },
    {
      "language": "vue",
      "autoFix": true
    }
  ]
}

ESLint 插件会自动检测项目目录下的 .eslintrc.js 文件, 并根据里面的设置进行格式化。

::: tip 注意 ESLint 插件保存时格式化的时候只会对当前文件有用。 :::

# 创建 eslintrc

全局安装 eslint 后, 可以使用 eslint --init 初始化 eslint 并创建 eslintrc

首先使用 npm 全局安装: npm install -g eslint

然后安装 VSCode 的 ESLint 插件, Ctrl + Shift + P 调出命令, 输入 ESLint: Create ESLint configuration

回答完毕后, 会在项目根下创建一个 .eslintrc.js 文件:

module.exports = {
  "env": {
    "browser": true,
    "commonjs": true,
    "es6": true,
    "node": true
  },
  "extends": "eslint:recommended",
  "parserOptions": {
    "ecmaFeatures": {
      "jsx": true
    },
    "ecmaVersion": 2018,
    "sourceType": "module"
  },
  "plugins": [
    "react"
  ],
  "rules": {
    "indent": [
      "error",
      2
    ],
    "linebreak-style": [
      "error",
      "unix"
    ],
    "quotes": [
      "error",
      "single"
    ],
    "semi": [
      "error",
      "never"
    ]
  }
};

# 使用第三方验证库

比如使用 vue 官方的 ESLint 验证库, 地址: https://github.com/vuejs/eslint-plugin-vue-libs

安装: yarn add -D eslint-plugin-vue-libs

在 .eslintrc.js 中配置:

{
  extends: ["plugin:vue-libs/recommended"],
  rules: {
    // override if necessary
  }
}

有几个受欢迎的eslint验证库可以配置:

  • standard
  • airbnb

.eslintrc.js 配置如下:

module.exports = {
  "extends": "airbnb"
};

# 与 Prettier 共存

在格式化代码方面， Prettier 确实和 ESLint 有重叠，但两者侧重点不同：ESLint 主要工作就是检查代码质量并给出提示，它所能提供的格式化功能很有限；而 Prettier 在格式化代码方面具有更大优势。而 Prettier 被设计为易于与 ESLint 集成，所以你可以轻松在项目中使两者，而无需担心冲突。

安装: yarn add -D eslint-plugin-prettier eslint-config-prettier prettier-eslint-cli

其公开了一个配置: plugin:prettier/recommended, 在 .eslintrc.js 中配置:

{
  "extends": [
    "plugin:prettier/recommended"
  ]
}

# 使用命令进行修复

如果需要手动修复, 可以在 package.json 中配置:

{
  "scripts": {
    "fix": "eslint --ext .js,.vue --fix --ignore-path .gitignore .",
    "vue-fix": "eslint src/pages/**/*.vue --fix",
    "format-code": "prettier-eslint --write \"src/**/*.js\" \"src/**/*.vue\""
  }
}

使用对应的命令即可修复:

$ yarn fix
$ yarn vue-fix
$ yarn format-code

跟 ESLint 插件的保存时自动格式化不同, 使用命令的话可以一次性格式化所有匹配的文件。
