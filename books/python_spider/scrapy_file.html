# Scrapy: 存取为文本

本文通过爬取 http://lab.scrapyd.cn 中的名言进行演示, 其部分HTML结构如下:

<div class="quote post">
	<span class="text">
		各种时代，各国诗人，个抓个的痒。scrapy中文网（http://www.scrapyd.cn）整理
	</span>
	<span>
		作者：<small class="author">木心<>
		<a href="http://lab.scrapyd.cn/archives/29.html">【详情】</a>
	</span>
	<p></p>
	<div class="tags">
		标签 ：
		<a href="http://lab.scrapyd.cn/tag/木心/">木心</a> ， 
		<a href="http://lab.scrapyd.cn/tag/艺术/">艺术</a>  
	</div>
</div>

页面由多个 .quote 构成, 每个 .quote 下包括 .text .author 以及 tags

# 单页数据过滤并保存

提取 http://lab.scrapyd.cn 首页的数据:

import scrapy
import codecs

class itemSpider(scrapy.Spider):
    name = 'listSpider'
    start_urls = ['http://lab.scrapyd.cn']

    def parse(self, response):
        mingyan = response.css('div.quote')  # 提取首页所有名言，保存至变量mingyan

        for v in mingyan:  # 循环获取每一条名言里面的：名言内容、作者、标签
            text = v.css('.text::text').extract_first()  # 提取名言
            autor = v.css('.author::text').extract_first()  # 提取作者
            tags = v.css('.tags .tag::text').extract()  # 提取标签
            tags = ','.join(tags)  # 数组转换为字符串

            # 接下来进行写文件操作，每个名人的名言储存在一个txt文档里面
            fileName = '%s-语录.txt' % autor  # 爬取的内容存入文件，文件名为：作者-语录.txt

            with codecs.open(fileName, 'a+', encoding='utf-8') as f:  # 不同人的名言保存在不同的txt文档，“a+”以追加的形式
                f.write('名言: ' + quotes + '\n')
                f.write('标签：' + tags + '\n')
                f.write('----------\n')
                f.close()

# 分页数据过滤并保存

分页导航结构:

<ol class='page-navigator'>
  ...
  <li class="next">
      <a href="http://lab.scrapyd.cn/page/2/">下一页 »</a>
  </li>
</ol>

提取 http://lab.scrapyd.cn 多个页面的数据:

import scrapy

class NextSpiderSpider(scrapy.Spider):
    name = "nextSpider"
    allowed_domains = ["lab.scrapyd.cn"]
    start_urls = ['http://lab.scrapyd.cn/']

    def parse(self, response):
        mingyan = response.css('div.quote')

        for v in mingyan:
            text = v.css('.text::text').extract_first()
            tags = ','.join(v.css('.tags .tag::text').extract())
            with open('语录.txt', "a+") as f:
                f.write('名言: ' + quotes + '\n')
                f.write('标签：' + tags + '\n')
                f.write('----------\n')
                f.close()

        # 判断下一页是否存在, 如果存在, 我们需要继续提交给 parse 执行关键看 scrapy 如何实现链接提交
        next_page = response.css('li.next a::attr(href)').extract_first()  # css选择器提取下一页链接
        if next_page is not None:  # 判断是否存在下一页
            next_page = response.urljoin(next_page)
            yield scrapy.Request(next_page, callback=self.parse)
  1. 前面部分的代码都一样, 增加了下一页是否为空的判断条件, 如果不为空, 则继续抓取
  2. response.uurljoin() 能替我们转换为绝对路径(也就是加上我们配置的域名)，如：/page/2/, 最终next_page转换为：http://lab.scrapyd.cn/page/2/
  3. scrapy.Request() 用于发起请求, 这里我们只使用了其两个参数:
    • 一个是我们继续爬取的链接（next_page），这里是下一页链接，当然也可以是内容页
    • 另一个是我们要把链接提交给哪一个函数(callback=self.parse)爬取，这里是parse函数，也就是本函数
    • 经过这么一个函数，下一页链接又提交给了parse，那就可以不断的爬取了，直到不存在下一页
    • 当然，我们也可以在下面另写一个函数，比如：内容页，专门处理内容页的数据

# 使用完整框架

以上示例, 所有逻辑均在爬虫文件中完成, 实际上, scrapy 的数据处理通常放于管道文件(pipelines.py), 这样, 框架的强大能力才得以显现, 不必所有操作均糅杂于一处。

# 定义管道

pipelines.py

class SpidertestPipeline(object):
    def process_item(self, item, spider):
        with open('语录.txt', 'a+', encoding='utf-8') as f:
            quotes = item['quote']
            tags = ','.join(item['tags'])  # 数组转换为字符串
            f.write('名言: ' + quotes + '\n')
            f.write('标签：' + tags + '\n')
            f.write('----------\n')
        return item

process_item 接收从爬虫主程序传递过来的 item 参数, 并进行加工处理, 处理完后返回到爬虫主程序继续执行。

定义了管道处理类及处理方法 process_item 后, 需要到 settings.py 文件进行注册:

ITEM_PIPELINES = {
   'spiderTest.pipelines.SpidertestPipeline': 300,
}

300的含义是执行优先级, 如果有多个管道, 则先自行数值较小的管道。
