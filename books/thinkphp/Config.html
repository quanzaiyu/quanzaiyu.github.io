# 配置文件

配置文件路径

在 ThinkPHP5.0 中，配置文件位于 application/config.php 中。 在 ThinkPHP5.1 中，配置文件位于 config 目录下。

配置格式

除了 .php 的配置文件外，ThinkPHP 还支持以下配置格式:

.xml、.json、.yaml、.ini

配置文件

在 ThinkPHP5.1 中，配置文件作用如下 :

配置文件名 描述
app.php 应用配置
cache.php 缓存配置
cookie.php Cookie配置
database.php 数据库配置
log.php 日志配置
session.php Session配置
template.php 模板引擎配置
trace.php 页面Trace配置
paginate.php 分页配置

# 扩展配置

扩展配置位于 application/extra 目录中，系统会默认加载里面的配置文件。

比如创建一个文件 application/extra/myconfig.php

return [
  'site_name' => '小昱个人网站',
  'site_author' => 'qzy'
];

在控制器的方法中获取，将文件名作为一级键值，跟上此文件下的配置名:

config('myconfig.site_name');

# 配置文件的优先级

ThinkPHP 默认会自动加载配置文件，但配置文件有加载优先级。

惯例配置 < 应用配置 < 扩展配置 < 场景配置 < 模块配置 < 动态配置

惯例配置

位于 thinkphp/convention.php

应用配置

位于 application/config.php

扩展配置

位于 application/extra 目录下

场景配置

首先在 application/config.php 中的 app_status 选项中指定场景，比如 :

app_status => 'dev'

则只需在 application 目录下新建一个同名的场景配置文件，即 dev.php 即可。

模块配置

比如在 application/index 模块下，新增一个 config.php 的配置文件，完整路径为 application/index/config.php。

动态配置

可以在控制器或模型中，通过 \think\Config::set() 进行配置。

# 配置作用域

相当于给配置加一个命名空间，用于配置的隔离。

加载配置

Config::load('配置文件的绝对路径', '配置项名称', '作用域的名称');

如

Config::load(APP_PATH . 'my_config.php', 'test', 'user');

其中 APP_PATH 定位会到 application 目录下。

再使用

Config::get('', '作用域名称');

加载此作用域下的所有配置。此时，所有的配置将加载到指定的配置项名称下，这里是 test。

# 优化配置文件

在 ThinkPHP5.0 中，配置文件没被拆分，可以通过在入口文件中使用 CONF_PATH 中指定配置文件存放的目录。

在 public/index.php 中定义 :

define('CONF_PATH', __DIR__, '/../config');

此时，我们需要在项目根目录下建立 config 文件夹，来管理配置文件。

默认情况下，配置文件目录与应用程序目录相同，即 application 下。

在 ThinkPHP5.1 中，不需要任何更改，config 目录已经分离。

# 读取配置

读取配置的几种方法:

方法一: 使用完整命名空间

\think\Config::get('配置名');

方法二: 使用助手函数

config('配置名');

两种方式都可直接获取指定名称的配置项，如果不带配置名参数，则获取所有配置，读取二级配置需要使用 . 分割。

ThinkPHP5.1 的配置参数全部采用二级配置的方式（默认一级配置为app），所以当你使用 config('name') 的时候其实相当于使用 config('app.name')

如

\think\Config::get('default_return_type');
\think\Config::get('database.type');

// 或使用 助手函数

config('default_return_type');
config('database.type');

# 读取某个一级配置的所有配置参数

读取某个一级配置的所有配置参数，可以使用以下几种方法:

Config::pull('app');

// 或者

Config::get('app.');

// 或者

config('app.');

# 判断某个配置项是否存在

\think\Config::has('配置名');

# 打印配置列表

方法一: 使用完整命名空间

namespace app\index\controller;

class Index {
  public function index () {
    $res = \think\Config::get();
    dump($res);
    return;
  }
}

方法二: 使用 use 加载命名空间

namespace app\index\controller;
use \think\Config;

class Index {
  public function index () {
    $res = Config::get();
    dump($res);
    return;
  }
}

# 动态配置

使用全局命名空间设置配置，只在当前作用于内有效

\think\Config::set('site_name', '小昱个人网站');
config('site_name'); // '小昱个人网站'

或者使用 助手函数 设置:

config('site_name', '小昱个人网站');

# 批量设置配置项

$config = ['order_no' => '1001', 'money' => 100];
\think\Config::set($config);

使用助手函数设置

$config = ['order_no' => '1001', 'money' => 100];
config($config);

# 指定默认输出格式

# html 输出

默认输出格式在 application/config.php 中进行设置，默认输出为 html:

'default_return_type' => 'html'

在控制器中

namespace app\index\controller;

class Index
{
  public function index()
  {
    return '<h1>Hello ThinkPHP</h1>';
  }
}

在浏览器中输入路径即可看到 HTML 格式的输出。

也可以直接通过 display() 函数输出 HTML 格式的数据:

$display = '<h1>Hello ThinkPHP</h1>';
return $this -> display($display);

或者使用 Response::create() 指定一个 html 类型的响应数据。

return Response::create('test','html');

# json 输出

'default_return_type' => 'json'

在 application/index/controller/index/Index.php 中

namespace app\index\controller;

class Index
{
  public function index()
  {
    $data = ['name'=>'quanzaiyu','gender'=>'male'];
    return ['data'=>$data,'code'=>1,'message'=>'操作完成'];
  }
}

在浏览器中输入路径即可看到 JSON 格式的输出。

也可以直接通过 json() 函数 代替 json_encode() 输出 JSON 格式的数据:

$data = ['name'=>'quanzaiyu','gender'=>'male'];
return json(['data'=>$data,'code'=>1,'message'=>'操作完成']);

或者使用 Response::create() 指定一个 json 类型的响应数据。

return Response::create(['msg' => '暂无数据'],'json');

# xml 输出

'default_return_type'    => 'xml'

在 application/index/controller/index/Index.php 中

namespace app\index\controller;

class Index
{
  public function index()
  {
    $data = ['name'=>'quanzaiyu','gender'=>'male'];
    return ['data'=>$data,'code'=>1,'message'=>'操作完成'];
  }
}

在浏览器中输入 http://www.my-tp5.com/ 即可看到 XML 格式的输出。

也可以直接通过 xml() 函数输出 XML 格式的数据:

$data = ['name'=>'quanzaiyu','gender'=>'male'];
return xml(['data'=>$data,'code'=>1,'message'=>'操作完成']);

或者使用 Response::create() 指定一个 xml 类型的响应数据。

return Response::create(['msg' => '暂无数据'],'xml');

# dump

在 TP 中，可以使用 dump() 替代 var_dump() 进行美化输出引用类型的数据，注意，只是输出数据不是返回数据。

如

public function index()
{
  $arr = ['a' => 0, 'b' => 2];
  dump($arr);
}
