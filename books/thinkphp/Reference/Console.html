# 参考手册: 命令行工具的使用

ThinkPHP5.0支持Console应用，使用如下命令输出支持的命令:

php think

子命令的调用规则为

php think 指令 参数

# 系统命令

指令 描述
build 自动生成目录和文件
help 帮助
list 指令列表
clear 清除缓存指令
make:controller 创建控制器文件
make:model 创建模型文件
optimize:autoload 生成类库映射文件
optimize:config 生成配置缓存文件
optimize:route 生成路由缓存文件
optimize:schema 生成数据表字段缓存文件

除此之外，还可以自己扩展指令。

# 生成模块

使用 php think build --module 模块 即可生成指定的模块

比如

php think build --module test

这样会生成一个 test 模块，并且生成必要的目录结构

test
  - controller # 控制器目录
    - index.php
  - model # 模型目录
  - view # 视图目录
  - common.php # 公共文件
  - config.php # 配置文件

# 生成控制器

使用 php think make:controller 控制器 默认可生成指定的资源控制器

可以使用 模块/控制器 生成:

php think make:controller index/Blog

也可以使用完整的 命名空间 生成:

php think make:controller app\index\controller\Blog

生成的控制器包括 index、create、save、read、edit、update、delete 这几个方法。

如果不想生成资源控制器，仅仅生成一个空白控制器，则可使用:

php think make:controller index\Blog --plain

# 生成模型类

使用 php think make:model 模型 即可生成指定的资源控制器

php think make:model index/Blog

# 清除缓存文件

使用 php think clear 即可清除 runtime 目录下的缓存文件。

也可指定要清除的目录:

php think clear --path d:\www\tp5\runtime\log\

# 创建自定义命令行

第一步，配置command.php文件，目录在application/command.php

return [
  'app\home\command\Test',
];

第二步，建立命令类文件，新建application/home/command/Test.php

namespace app\home\command;

use think\console\Command;
use think\console\Input;
use think\console\Output;

class Test extends Command
{
  protected function configure()
  {
    $this->setName('test')->setDescription('Here is the remark ');
  }

  protected function execute(Input $input, Output $output)
  {
    $output->writeln("TestCommand:");
  }
}

这个文件定义了一个叫test的命令，备注为Here is the remark，执行命令会输出TestCommand。

第三步，测试-命令帮助-命令行下运行

php think

输出
