# 使用 Composer 安装

Composer中文网: https://www.phpcomposer.com/

$ composer create-project topthink/think
$ composer create-project topthink/think=5.0.* tp5  --prefer-dist

如果提示错误 [Composer\Exception\NoSslException] The openssl extension is required for SSL/TLS protection but is not available.，只需打开 PHP 扩展 openssl 即可。

如果提示 :

即为安装成功

# 使用 Git 安装

GitHub

  • 应用项目：https://github.com/top-think/think
  • 核心框架：https://github.com/top-think/framework

码云

  • 应用项目：https://git.oschina.net/liu21st/thinkphp5.git
  • 核心框架：https://git.oschina.net/liu21st/framework.git

Coding

  • 应用项目：https://git.coding.net/liu21st/thinkphp5.git
  • 核心框架：https://git.coding.net/liu21st/framework.git

首先克隆下载应用项目仓库

$ git clone https://github.com/top-think/think tp5

然后切换到tp5目录下面，再克隆核心框架仓库：

$ git clone https://github.com/top-think/framework thinkphp

# 配置虚拟主机

首先修改 hosts :

127.0.0.1       www.my-tp5.com

# Apache

如果服务器是 Apache，则修改 httpd-vhosts.conf :

<VirtualHost *:80>
  ServerAdmin webmaster@tp5.com
  DocumentRoot "D:\\projects\\think\\public"
  ServerName tp5
  ServerAlias www.my-tp5.com
  DirectoryIndex news.html index.html index.htm index.php
  ErrorLog "logs/tp5-error.log"
  CustomLog "logs/tp5-access.log" common
</VirtualHost>

或者修改 vhosts.conf :

<VirtualHost *:80>
    DocumentRoot "D:\projects\think\public"
    ServerName www.my-tp5.com
    ServerAlias
  <Directory "D:\projects\think\public">
      Options FollowSymLinks ExecCGI
      AllowOverride All
      Order allow,deny
      Allow from all
      Require all granted
  </Directory>
</VirtualHost>

# Nginx

如果服务器是 Nginx，则修改 vhosts.conf :

server {
    listen       80;
    server_name  www.my-tp5.com ;
    root   "D:/projects/think/public";
    location / {
        index  index.html index.htm index.php;
    }
}

一切配置就绪，只需在浏览器中输入 http://www.my-tp5.com 即可。

# URL 访问

在默认情况下，在 URL 中访问对应资源通过 PATH_INFO 的方式访问，格式如下 :

http://host/index.php/模块/控制器/操作

比如在 index 模块下，有一个 Demo 控制器，里面有一个方法 :

application/index/controller/Demo.php

public function getName ($name = 'qzy') {
  return 'Hello ' . $name;
}

此时在浏览器中输入 http://www.my-tp5.com/index.php/index/Demo/getName 会看到输出 Hello qzy 。

也可使用传统的MVC兼容访问模式:

http://www.my-tp5.com/?s=/index/Demo/getName
// or
http://www.my-tp5.com/?s=index/Demo/getName

# URL 传值

在上述方法中，可以看到 getName 方法接受一个 $name 参数，如果要传入参数，则在 URL 中指定，格式为:

http://host/index.php/模块/控制器/操作/key/value

比如上面的例子可以这样传值:

http://www.my-tp5.com/index.php/index/Demo/getName/name/Quan

可以看到输出 Hello Quan。

对应关系如下:

不过，URL 传值只能传递字符串、数值，使用 GET 方式来传值到类中方法。

PATH_INFO 也支持查询字符串传值:

http://www.my-tp5.com/index.php/index/Demo/getName?name=5

传统 MVC 兼容模式传参格式为:

http://www.my-tp5.com/?s=index/Demo/getName&name=5

# 多个参数传递

比如刚才控制器中方法包含多个参数:

application/index/controller/Demo.php

namespace app\index\controller;
class Demo {
  public function say ($name, $age) {
    return 'My name is  ' . $name . ', I am ' . $age . 'years old.';
  }
}

key-value 键值对写法

http://host/index.php/模块/控制器/操作/key1/value1/key2/value2

比如:

http://www.my-tp5.com/index.php/index/Demo/say/name/Quan/age/18

也可以颠倒顺序，但必须是 key-value 配对

http://www.my-tp5.com/index.php/index/Demo/say/age/18/name/Quan

PATH_INFO 查询字符串传值的写法

http://www.my-tp5.com/index.php/index/Demo/say?name=5&age=18

传统 MVC 兼容模式传参格式

http://www.my-tp5.com/?s=index/Demo/say&name=5&age=18

# 配置传参规则

在 config/app.php 配置文件中，可以看到那么一个配置:

// URL参数方式 0 按名称成对解析 1 按顺序解析
'url_param_type' => 0,

这里，默认为 0，按照 key-value 的方式解析路由，如果是 1，则按照顺序解析，上面的路径必须写为:

http://www.my-tp5.com/index.php/index/Demo/say/Quan/18

# URL 重写

可以通过URL重写隐藏应用的入口文件index.php。

# Nginx

在虚拟主机配置文件中加入:

if (!-e $request_filename) {
  rewrite  ^(.*)$  /index.php?s=/$1  last;
}

完整配置如下:

server {
  listen       80;
  server_name  www.my-tp5.com ;
  root   "D:/projects/think/public";
  location / {
    index  index.html index.htm index.php;
    if (!-e $request_filename) {
      rewrite  ^(.*)$  /index.php?s=/$1  last;
    }
  }
}

也可使用 rewrite ^/(.*)$ /index.php/$1 last; 进行重写。

此时，就可省略 index.php 直接通过 PATH_INFO 访问:

http://www.my-tp5.com/模块/控制器/操作

# Apache

  1. httpd.conf 配置文件中加载了 mod_rewrite.so 模块
  2. AllowOverride None 将 None 改为 All (可能是 vhosts.conf 配置的虚拟主机中)
  3. 把下面的内容保存为.htaccess文件放到应用入口文件的同级目录下
<IfModule mod_rewrite.c>
  Options +FollowSymlinks -Multiviews
  RewriteEngine On

  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteRule ^(.*)$ index.php [L,E=PATH_INFO:$1]
</IfModule>

# 隐藏模块

如果整个系统中只有一个 index 模块，想要隐藏此模块的名称，则只需要在 public/index.php 中改写 :

Container::get('app')->bind('index')->run()->send();

此时，访问只需要通过:

http://serverName/index.php/控制器/操作

如果通过 URL 重写后，只需要:

http://serverName/控制器/操作

# 隐藏控制器

如果整个系统相当简单，只有一个模块和一个控制器，还可以隐藏控制器:

Container::get('app')->bind('index/index')->run()->send();

此时就更加简单，访问只需 http://serverName/操作 即可。
