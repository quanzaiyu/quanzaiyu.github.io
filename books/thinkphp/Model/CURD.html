# 模型: 增删改查

# 查询数据: 获取单条数据

# get

直接使用 get() 即可获取指定主键中的数据。

$data = User::get(12);

与下面操作相同 :

$data = User::where('id', 12)->find();

转化为 SQL 为

SELECT * FROM `zh_user` WHERE `id` = 12 LIMIT 1

# 查询数据: 获取多条数据

# all

使用 all 方法，可以获取模型中的所有数据 :

$data = User::all();

可以在 all 中传入一个数组，获取指定主键的数据 :

$data = User::all([1,2,3]);

与下面操作相同 :

$data = User::where('id', 'in', '1,2,3')->select();

转化为 SQL 为:

SELECT * FROM `user` WHERE `id` IN (1,2,3)

# 查询数据: find 与 select

  • find 返回查找到的第一条数据，若无符合的数据返回 NULL，通常配合 get 使用
  • select 返回查找到的所有数据， 若无符合条件的数据返回 array(0) {}，通常配合 where 使用

# 查询数据: 获取某个字段或某列的值

// 获取某个用户的积分
User::where('id',10)->value('score');
// 获取某个列的所有值
User::where('status',1)->column('name');
// 以id为索引
User::where('status',1)->column('name','id');

::: warning value和column方法返回的不再是一个模型对象实例，而是纯粹的值或者某个列的数组。 :::

# 添加数据: 添加单条数据

方式一: 在模型实例中指定要添加的数据

首先实例化模型，指定要添加的字段后调用 save 方法;

$user = new User;
$user->name = 'xiaoyu';
$user->email = 'xiaoyu@xiaoyulive.top';
$user->save();
// 获取自增ID
echo $user->id;

::: tip 提示 save 方法新增数据返回的是写入的记录数（通常是1），而不是自增主键值。但是可以在调用 save 方法后获取其它自动更新的字段。 :::

方式二: 在 save 方法调用的时候指定

$user = new User;
$user->save([
  'name'  =>  'xiaoyu',
  'email' =>  'xiaoyu@xiaoyulive.top'
]);

方式三: 在实例化的时候传入

$user = new User([
  'name'  =>  'xiaoyu',
  'email' =>  'xiaoyu@xiaoyulive.top'
]);
$user->save();

方式四: 使用 data() 赋值

$user = new User;
$user->data([
  'name'  =>  'xiaoyu',
  'email' =>  'xiaoyu@xiaoyulive.top'
]);
$user->save();

以上代码翻译成 SQL 为:

INSERT INTO `user` (`name` , `email`) VALUES ('xiaoyu' , 'xiaoyu@xiaoyulive.top')

# 添加数据: 添加多条数据

$user = new User;
$list = [
  ['name'  =>  'xiaoyu', 'email' =>  'xiaoyu@xiaoyulive.top'],
  ['name'=> 'yu','email'=>'yu@qq.com']
];
$user->saveAll($list);

::: tip 提示 saveAll 方法新增数据返回的是包含新增模型（带自增 ID）的数据集对象。 :::

# 添加数据: 静态写入

直接静态调用 create 方法创建并写入：

$user = User::create([
  'name'  =>  'xiaoyu',
  'email' =>  'xiaoyu@xiaoyulive.top'
]);
echo $user->name;
echo $user->email;
echo $user->id; // 获取自增ID

::: tip 和 save 方法不同的是，create 方法返回的是当前模型的对象实例。 :::

create方法的第二个参数可以传入允许写入的字段列表（传入 true 则表示仅允许写入数据表定义的字段数据），例如：

// 只允许写入 name 和 email 字段的数据
$user = User::create([
  'name'  =>  'xiaoyu',
  'email' =>  'xiaoyu@xiaoyulive.top'
], ['name', 'email']);
echo $user->name;
echo $user->email;
echo $user->id; // 获取自增ID

# 添加数据: 从请求参数中获取并过滤

如果需要从请求参数中获取数据，并过滤非数据表字段的数据:

$user = new User;
// 过滤 get 参数中的非数据表字段数据
$user->allowField(true)->save($_GET);

比如在地址栏输入

http://www.my-tp5.com/hello?name=quanzaiyu&email=yu@xiaoyulive.top

转换为 SQL 为

INSERT INTO `zh_user` (`name` , `email`) VALUES ('quanzaiyu' , 'yu@xiaoyulive.top')

指定某些字段写入

$user = new User;
// post数组中只有 name 和 email 字段会写入
$user->allowField(['name', 'email'])->save($_GET);

最佳的建议是模型数据赋值之前就进行数据过滤

$user = new User;
// 过滤 post 数组中的非数据表字段数据
$data = Request::only(['name','email']);
$user->save($data);

# 更新数据: 查找并更新

# 通过主键查询数据并更新

在取出数据后，更改字段内容后使用save方法更新数据。这种方式是最佳的更新方式。

$user = User::get(1);
$user->name = 'xiaoyu';
$user->email = 'xiaoyu@xiaoyu.top';
$user->save();

# 使用查询构造器查询数据并更新

$user = User::where('status',1)
	->where('name','liuchen')
	->find();
$user->name = 'xiaoyu';
$user->email = 'xiaoyu@xiaoyu.top';
$user->save();

save 方法更新数据，只会更新变化的数据，对于没有变化的数据是不会进行重新更新的。如果需要强制更新数据，可以使用下面的方法：

$user->force()->save();

# 直接更新数据

使用 save 的第二个参数可以指定要更新的条件

$user = new User;
// save 方法第二个参数为更新条件
$user->save([
  'name'  => 'yu',
  'email' => 'yu@qq.com'
],['id' => 12]);

# 接收表单数据并过滤数据更新

$user = new User;
// 过滤 post 数组中的非数据表字段数据
$user->allowField(true)->save($_POST,['id' => 1]);

如果需要指定更新某些字段 :

$user = new User();
// post 数组中只有 name 和 email 字段会写入
$user->allowField(['name','email'])->save($_POST, ['id' => 1]);

最佳实践: 在获取数据的时候过滤数据:

$user = new User();
// post 数组中只有 name 和 email 字段会写入
$data = Request::only(['name','email']);
$user->save($data, ['id' => 1]);

# 更新数据: 批量更新

使用 saveAll 方法批量更新数据，只需要在批量更新的数据中包含主键即可

$user = new User;
$list = [
  ['id'=>1, 'name'=>'yu', 'email'=>'yu@qq.com'],
  ['id'=>2, 'name'=>'quan', 'email'=>'quan@qq.com']
];
$user->saveAll($list);

# 更新数据: 静态方法更新

使用模型的静态 update 方法更新

User::where('id', 1)
  ->update(['name' => 'yu']);

或者

User::update(['id' => 1, 'name' => 'yu']);

转化为 SQL 为

UPDATE `user` SET `name` = 'yu' WHERE `id` = 1

# 更新数据: 只读字段

只读字段用来保护某些特殊的字段值不被更改，这个字段的值一旦写入，就无法更改。 要使用只读字段的功能，我们只需要在模型中定义 readonly 属性

protected $readonly = ['name', 'email'];

5.1 版本支持动态设置只读字段 :

$user = User::get(5);
// 更改某些字段的值
$user->name = 'XXYu';
$user->email = 'XXYu@gmail.com';
$user->nickname = '小小昱';
// 保存更改后的用户数据
$user->readonly(['name', 'email'])->save();

这里，我们对 name 和 email 字段设置了只读，因此只有 nickname 字段的值被更新了，而 name 和 email 的值仍然还是更新之前的值。

只读字段仅针对模型的更新方法，如果使用数据库的更新方法则无效，例如下面的方式无效。

$user = new User;
 // 要更改字段值
$data['name'] = 'XXYu';
$data['email'] = 'XXYu@gmail.com';
$data['nickname'] = '小小昱';
 // 保存更改后的用户数据
$user->where('id', 5)->update($data);

# 更新数据: 自动识别

我们已经看到，模型的新增和更新方法都是save方法，系统有一套默认的规则来识别当前的数据需要更新还是新增。

  • 实例化模型后调用save方法表示新增；
  • 查询数据后调用save方法表示更新；
  • save方法传入更新条件后表示更新；

如果你的数据操作比较复杂，可以用isUpdate方法显式的指定当前调用save方法是新增操作还是更新操作。

显式更新数据：

// 实例化模型
$user = new User;
// 显式指定更新数据操作
$user->isUpdate(true)
  ->save(['id' => 1, 'name' => 'thinkphp']);

显式新增数据：

$user = User::get(1);
$user->name = 'thinkphp';
// 显式指定当前操作为新增操作
$user->isUpdate(false)->save();

不要在一个模型实例里面做多次更新，会导致部分重复数据不再更新，正确的方式应该是先查询后更新或者使用模型类的update方法更新。

::: danger 如果你调用save方法进行多次数据写入的时候，需要注意，第二次save方法的时候必须使用isUpdate(false)，否则会视为更新数据。 :::

::: tip 最佳实践 更新的最佳实践原则是： 如果需要使用模型事件，那么就先查询后更新，如果不需要使用事件，直接使用静态的Update方法进行条件更新，如非必要，尽量不要使用批量更新。 :::

# 删除数据: 根据主键删除

使用 delete 先查找再删除

$user = User::get(17);
$user->delete();

转化为 SQL 为

DELETE FROM `user` WHERE `id` = 17

或者直接调用静态方法 destroy

User::destroy(1);
// 支持批量删除多个数据
User::destroy('1,2,3');
// 或者
User::destroy([1,2,3]);

::: danger 当 destroy 方法传入空值（包括空字符串和空数组）的时候不会做任何的数据删除操作，但传入0则是有效的 :::

# 删除数据: 条件删除

还支持使用闭包删除，例如：

User::destroy(function($query){
  $query->where('id','>',10);
});

或者通过数据库类的查询条件删除

User::where('id','>',10)->delete();

::: danger 直接调用数据库的 delete 方法的话无法调用模型事件。 :::

::: tip 最佳实践 删除的最佳实践原则是：如果删除当前模型数据，用delete方法，如果需要直接删除数据，使用destroy静态方法。 :::

# 删除数据: 软删除

在实际项目中，对数据频繁使用删除操作会导致性能问题，软删除的作用就是把数据加上删除标记，而不是真正的删除，同时也便于需要的时候进行数据的恢复。

要使用软删除功能，需要引入SoftDelete trait，例如User模型按照下面的定义就可以使用软删除功能：

namespace app\index\model;

use think\Model;
use think\model\concern\SoftDelete;

class User extends Model
{
  use SoftDelete;
  protected $deleteTime = 'delete_time';
}

deleteTime 属性用于定义你的软删除标记字段，ThinkPHP 的软删除功能使用时间戳类型（数据表默认值为Null），用于记录数据的删除时间。

::: tip 软删除字段数据库中默认应为 int 类型，长度为10，如果使用 datetime 或 timestamp 类型，需要在程序中指定其类型:

protected $type = [
  'delete_time'=>'datetime',
];

// or

protected $type = [
  'delete_time'=>'timestamp',
];

:::

定义好模型后，我们就可以使用：

// 软删除
User::destroy(1);
// 真实删除
User::destroy(1,true);

$user = User::get(1);
// 软删除
$user->delete();
// 真实删除
$user->delete(true);

默认情况下查询的数据不包含软删除数据，如果需要包含软删除的数据，可以使用下面的方式查询：

User::withTrashed()->find();
User::withTrashed()->select();

如果仅仅需要查询软删除的数据，可以使用：

User::onlyTrashed()->find();
User::onlyTrashed()->select();

恢复被软删除的数据

$user = User::onlyTrashed()->find(1);
$user->restore();

::: warning 软删除仅对模型的删除方法有效，如果直接使用数据库的删除方法则无效，例如下面的方式无效（将不会执行任何操作）。 :::

$user = new User;
$user->where('id',1)->delete();
