# 小程序跳转

# 跳转到另一个小程序

使用 wx.navigateToMiniProgram(Object object) 可进行小程序之间的跳转：

wx.navigateToMiniProgram({
  appId: 'appid',
  path: 'page/index/index',
  extraData: {
    foo: 'bar'
  },
  envVersion: 'develop',
  success(res) {
    // 打开成功
  }
})

其中参数：

  • appId 将要跳转的小程序 appId
  • path 跳转之后小程序的路径
  • extraData 要传递的数据，目标小程序可在 App.onLaunch，App.onShow 中获取到这份数据
  • envVersion 要打开的小程序版本，默认 release，可选 develop、trial、release 三个值

使用限制：

  • 需要用户触发跳转
  • 需要用户确认跳转
  • 每个小程序可跳转的其他小程序数量限制为不超过 10 个

要跳转的小程序列表需要在 app.json 中进行配置：

{
  "navigateToMiniProgramAppIdList": ["appid1", "appid2"]
}

# 返回跳转前的小程序

wx.navigateBackMiniProgram(Object object)，返回到上一个小程序。只有在当前小程序是被其他小程序打开时可以调用成功。

wx.navigateBackMiniProgram({
  extraData: {
    foo: 'bar'
  },
  success (res) {
    // 返回成功
  }
})

# 跳转到网页

# 小程序

在需要跳转的地方：

wx.navigateTo({url: '/pages/webview?url=' + encodeURIComponent('h5页面的 url')});

pages/webview.wxml

<web-view src="{{url}}" bindmessage="handlePostMessage"></web-view>

pages/webview.js

在生命周期函数中：

onLoad: function (options) {
 this.setData({
   url: decodeURIComponent(options.url),
 });

# uni-app

如果是 uni-app，则更加简单。

首先，创建一个 webview 容器，用来容纳任意网页：

<template>
  <web-view :src="url" @message="handlePostMessage"></web-view>
</template>

<script>
export default {
  data () {
    return {
      url: ''
    }
  },
  onLoad (args) {
    this.url = args.url
  },
  methods: {
    // 接收 h5 页面传递过来的参数
    handlePostMessage: function (e) {
      const data = e.detail;
      console.log(data);
    }
  }
}
</script>

# 网页返回到小程序

# 小程序

<web-view/>网页中可使用JSSDK 1.3.2提供的接口返回小程序页面。

首先，在网页中引入：

<script type="text/javascript" src="https://res.wx.qq.com/open/js/jweixin-1.3.2.js"></script>
1

在需要触发跳转的地方填入：

wx.miniProgram.getEnv(function (res) {
  if (res.miniprogram) {
    wx.miniProgram.navigateTo({url: '/pages/home/home'});
    wx.miniProgram.postMessage({data: {id: '1234'}}); // 传的参数
  }
});
1
2
3
4
5
6

wx.miniProgram 的用法跟下程序中路由写法一致，参见 web-view、weixin-js-sdk、wx.navigateTo

支持的接口有：

  • wx.miniProgram.navigateTo 参数与小程序接口一致
  • wx.miniProgram.navigateBack 参数与小程序接口一致
  • wx.miniProgram.switchTab 参数与小程序接口一致
  • wx.miniProgram.reLaunch 参数与小程序接口一致
  • wx.miniProgram.redirectTo 参数与小程序接口一致
  • wx.miniProgram.postMessage 向小程序发送消息
  • wx.miniProgram.getEnv 获取当前环境

无论是小程序跳转H5页面还是跳转回来都是可以传递参数的，传参的方式有两种

  1. 直接拼接在跳转链接的后面。跳转到小程序页面在目标页面的onload的options中获取传递过来的参数即可，跳转到网页直接从查询字符串中获取即可。
  2. 发送方通过 postMessage 传递消息，对应了接收方的中的 handlePostMessage。
