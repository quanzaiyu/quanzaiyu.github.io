router-link vs router.push

声明式编程式
<router-link :to="...">router.push(...)

声明式路由 router-link

在使用 vue-router 的时候，可以使用标签跳转的形式，即 router-link，配置其 to 属性，决定跳转的页面

<router-link to="/foo">Go to Foo</router-link>
<router-link to="/bar">Go to Bar</router-link>

编程式路由 router.push

当然，如果不方便使用标签，也可以使用 js 控制跳转，即 router.push

格式:

router.push(location, onComplete?, onAbort?)
1

在组件中使用如下:

<div @click="go('/foo')"></div>
<div @click="go('/bar')"></div>

router.push 有以下几种格式:

// 字符串
router.push('home')

// 对象
router.push({ path: 'home' })

// 命名的路由
router.push({ name: 'user', params: { userId: 123 }})

// 带查询参数，变成 /register?plan=private
router.push({ path: 'register', query: { plan: 'private' }})

注意

如果提供了 path，params 会被忽略，上述例子中的 query 并不属于这种情况。

以下两种写法相同:

const userId = 123
router.push({ name: 'user', params: { userId }}) // -> /user/123
router.push({ path: `/user/${userId}` }) // -> /user/123

// 错误的写法: 这里的 params 不生效
router.push({ path: '/user', params: { userId }}) // -> /user

路由历史记录方法

在使用路由的单页应用中，每访问一个页面，便将其加入路由历史栈当中，之前提到过的 push 方法就是这个原理。

除了 push 方法，vue-router 还提供其他控制路由的方法，操作方法同 window.history。

router.back

退回到上一个路由历史栈中的页面。

this.$router.back() // 后退
1

router.forward

前进到下一个路由历史栈中的页面。

this.$router.forward() // 前进
1

router.go(n)

直接抵达指定的路由历史栈中的某个页面，指定n值，当前页面为 0，正数为此页之后的页面，负数为此页之前的页面。

// 在历史记录中前进一步，等同于 history.forward()
router.go(1)

// 在历史记录中后退一步，等同于 history.back()
router.go(-1)
1
2
3
4
5

router.replace

格式:

router.replace(location, onComplete?, onAbort?)
1

跟 router.push 用法相同，但它不会向 history 添加新记录，而是替换掉当前的 路由历史栈 记录。

声明式编程式
<router-link :to="..." replace>router.replace(...)

HTML5 History 模式

vue-router 默认 hash 模式 —— 使用 URL 的 hash 来模拟一个完整的 URL，于是当 URL 改变时，页面不会重新加载。

如果不想要很丑的 hash，我们可以用路由的 history 模式，这种模式充分利用 history.pushState API 来完成 URL 跳转而无须重新加载页面。

const router = new VueRouter({
  mode: 'history',
  routes: [...]
})
1
2
3
4

当你使用 history 模式时，URL 就像正常的 url，例如 http://yoursite.com/user/id，也好看！

不过这种模式要玩好，还需要后台配置支持。因为我们的应用是个单页客户端应用，如果后台没有正确的配置，当用户在浏览器直接访问 http://oursite.com/user/id 就会返回 404，这就不好看了。

所以呢，你要在服务端增加一个覆盖所有情况的候选资源：如果 URL 匹配不到任何静态资源，则应该返回同一个 index.html 页面，这个页面就是你 app 依赖的页面。

后端配置

Apache

<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>

http.createServer((req, res) => {
  fs.readFile('index.htm', 'utf-8', (err, content) => {
    if (err) {

基于 Node.js 的 Express

对于 Node.js/Express，请考虑使用 connect-history-api-fallback 中间件。

Internet Information Services (IIS)

  1. 安装 IIS UrlRewrite
  2. 在你的网站根目录中创建一个 web.config 文件，内容如下：
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
  <system.webServer>
    <rewrite>
      <rules>
        <rule name="Handle History Mode and custom 404/500" stopProcessing="true">
          <match url="(.*)" />
          <conditions logicalGrouping="MatchAll">
            <add input="{REQUEST_FILENAME}" matchType="IsFile" negate="true" />
            <add input="{REQUEST_FILENAME}" matchType="IsDirectory" negate="true" />
          </conditions>
          <action type="Rewrite" url="/" />
        </rule>
      </rules>
    </rewrite>
  </system.webServer>
</configuration>

404

这么做以后，服务器就不再返回 404 错误页面，因为对于所有路径都会返回 index.html 文件。为了避免这种情况，你应该在 Vue 应用里面覆盖所有的路由情况，然后在给出一个 404 页面。

const router = new VueRouter({
  mode: 'history',
  routes: [
    { path: '*', component: NotFoundComponent }
  ]
})
