安装与使用

vue本身并不强制用户使用脚手架进行安装，用户完全可以自定义进行安装，不过为了方便起见，脚手架中已集成各种必要的配置，我们可以站在巨人的肩膀上，将重心放到开发上，而不是搭建项目上折腾。

vue-cli

相关链接:

GitHub : vue-cli、npm : vue-cli

下载 vue-cli

npm install -g vue-cli
1

初始化一个项目

vue init <template-name> <project-name>
1

比如

vue init webpack my-project
1

官方 Templates

官方还提供其他一些常用的模板，webpack 只是其中一种 :

  • webpack - A full-featured Webpack + vue-loader setup with hot reload, linting, testing & css extraction.
  • webpack-simple - A simple Webpack + vue-loader setup for quick prototyping.
  • browserify - A full-featured Browserify + vueify setup with hot-reload, linting & unit testing.
  • browserify-simple - A simple Browserify + vueify setup for quick prototyping.
  • pwa - PWA template for vue-cli based on the webpack template
  • simple - The simplest possible Vue setup in a single HTML file

非官方 Template

vue init username/repo#<branch-name> my-project
1

其中 username/repo 分别代表 GitHub 用户名和其放置脚手架的仓库，branch-name 代表分支名称，可以省略默认为master。

本地 Template

vue init ~/fs/path/to-custom-template my-project
1

用户自己存放于本地的脚手架仍然可以通过本地路径的方式获取并创建项目。
